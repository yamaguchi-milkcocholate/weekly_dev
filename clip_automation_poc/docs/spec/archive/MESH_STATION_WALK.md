# 仕様書：東京23区高解像度不動産可視化用メッシュマスター作成

## 1. 目的

不動産取引価格情報の詳細な位置情報不足を補完するため、東京23区を100m四方のメッシュで区切り、各メッシュ中心点における「市区町村名」「地区名」「最寄駅」「徒歩分数」を事前に計算したルックアップテーブルを作成する。

## 2. システム要件

- **言語**: Python 3.10+
- **主要ライブラリ**: `pandas`, `geopandas`, `shapely`, `scipy` (cKDTreeを使用)
- **出力形式**: CSVファイル (`tokyo_23_mesh_master.csv`) および GeoParquet形式（推奨）

## 3. データソース

1. **駅座標データ**: 国土数値情報（鉄道データ）または駅データ.jp等のAPI/CSV。
2. **行政区域ポリゴン**: 国土数値情報「行政区域」または「町丁・字等別境界データ（世界測地系）」。
3. **23区境界**: 東京都23区に限定するためのフィルタリング。

## 4. 処理ロジック詳細

### Step 1: 空間メッシュの生成

- 東京23区のバウンディングボックスを取得し、100m間隔（約0.0009度/0.0011度）の格子点を生成する。
- 生成した点に対して、東京23区の行政ポリゴンで `sjoin` を行い、区外の点を除去する。

### Step 2: 行政属性の付与

- 各メッシュ中心点に対し、以下の属性を付与する。
- `市区町村名` (City)
- `地区名` (District / Town name)

### Step 3: 最寄駅・徒歩分数の計算（cKDTreeによる高速検索）

1. 23区内および境界付近（+5km）の全駅の緯度経度をk-d木に登録する。
2. 各メッシュ中心点から最も近い駅を検索。
3. **距離の計算**: 大圏距離（Haversine）を用い、駅からの直線距離（メートル）を算出。
4. **徒歩分数の変換**: `距離(m) / 80` で算出し、小数点第1位で保持する。

### Step 4: データ整合性のバリデーション

- 不動産取引データの `地区名` とメッシュマスターの `地区名` が紐づくよう、表記揺れ（例：ケ・ヶ、一丁目・1丁目）の正規化ロジックを組み込む。

## 5. 出力データ構造 (Schema)

| Column Name     | Type   | Description               |
| --------------- | ------ | ------------------------- |
| `mesh_id`       | String | ユニークなメッシュID      |
| `latitude`      | Float  | メッシュ中心の緯度        |
| `longitude`     | Float  | メッシュ中心の経度        |
| `city_name`     | String | 市区町村名 (例: 千代田区) |
| `district_name` | String | 地区名 (例: 飯田橋)       |
| `station_name`  | String | 最寄駅名 (例: 飯田橋駅)   |
| `distance_m`    | Float  | 駅からの直線距離          |
| `walk_minutes`  | Float  | 徒歩分数 (distance / 80)  |

## 6. 実装上の注意点

- **座標系**: 全ての計算は世界測地系（WGS84 / EPSG:4326）で行い、距離計算時のみ平面直角座標系（EPSG:6677）へ投影するかHaversine式を用いること。
- **パフォーマンス**: 23区を100mメッシュで埋めると約6万〜8万点となる。`apply` ループは避け、ベクトル化演算またはcKDTreeを使用すること。
