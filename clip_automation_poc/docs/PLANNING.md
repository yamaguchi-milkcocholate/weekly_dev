# 企画書：都市のダイナミズム - 不動産データの可視化による「現象」の提示

## 1. プロジェクトの理念（Why）

不動産市場は、日本の経済・社会の変化が最も色濃く反映される場でありながら、その情報は断片的で、一般層には「見えにくい」ものとなっている。
本プロジェクトは、国土交通省のオープンデータをエンジニアリングの力で「直感的な現象」へと変換し、**「都市は生き物のように変容し続けている」という客観的事実**を提示する。視聴者に解釈を委ねることで、質の高い議論と街への知的好奇心を喚起することを目的とする。

## 2. ターゲットと視聴者像（Who）

- **メイン:** 30代〜50代の一般ビジネス層・投資家層。
- **ニーズ:** 経済の大きな流れを、複雑な表ではなく、直感的に短時間で理解したい。
- **動機:** 自身の資産形成やキャリア形成において、都市の将来性を判断する「物差し」を求めている。

## 3. コンテンツ・コンセプト（What）

### メインタイトル案

**「Urban Dynamics: [Area Name] 1996 - 2026」**
〜地価の地殻変動から読み解く、都市の興亡〜

### シリーズ構成

1. **Metropolis Trace:** 東京、大阪、名古屋といった大都市圏の30年間の推移。
2. **Redevelopment Impact:** 虎ノ門や武蔵小杉など、特定の再開発が周囲を「侵食」する様子。
3. **Demographic Shift:** 人口流入と地価上昇の相関を、2つのレイヤーで重ねて見せる。

## 4. 映像表現の仕様（Visual Direction）

「データが自ら語る」ような、洗練されたプロフェッショナルな美学を追求する。

- **Color Palette:** ダークネイビーを基調とした背景に、地価上昇を「シアン〜マゼンタ」の蛍光グラデーションで表現。
- **Camera Work:** \* **低空ドローン視点:** 3D化された地価の「柱」の間を滑らかに縫うように移動。
- **マクロ・マイクロの往復:** 日本全体から、特定の交差点まで一気にズームインする「ワープ」演出。

- **UI Elements:** 画面隅でカウントアップする「西暦」、分析ソースを示す「Log Window」、データの信頼区間を示す淡い「Halo（光輪）」エフェクト。

## 5. 技術スタックと制作フロー（How）

### データパイプライン

1. **Extract:** Python（Polars）を用いて不動産情報ライブラリから公示地価・人口・都市計画データを抽出。
2. **Analyze:** 線形回帰およびヘドニック法を用い、欠損値の補完と代表物件スペックの価格推移を算出。
3. **Visualize:** `pydeck` (Deck.gl) または `PyVista` で3Dアニメーションのベース動画を生成。
4. **Enhance:** ビデオ生成AI（Runway Gen-3等）による質感のアップグレード（Video-to-Video）。

**境界GeoJSONソース:** 自治体の境界データは、JapanCityGeoJson（GitHub）を利用。
https://github.com/niiyz/JapanCityGeoJson

### 駅データソース（組み合わせ利用）

駅情報は単一ソースに依存せず、用途に応じて以下を組み合わせて利用する。

1. **駅データ.jp（CSV）**
   - 特徴: 緯度経度に加えて路線情報・接続駅情報が整理されている
   - 入手方法: 無料会員登録で最新データをダウンロード（商用利用は別途ライセンス確認）
   - URL: https://www.ekidata.jp/

2. **公共交通オープンデータセンター（API）**
   - 特徴: JSON-LD/GeoJSONで駅施設・出口など詳細情報を取得可能
   - 入手方法: 開発者アカウント作成
   - URL: https://www.odpt.org/

3. **国土数値情報（GISデータ）**
   - 特徴: 政府提供の公式データ。鉄道レイヤーで地図描画や統計解析向き
   - 入手方法: 国土数値情報ダウンロードサービスから取得
   - URL: https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-N02-v3_1.html

#### データソース比較

| データソース名             | 更新頻度     | 形式の扱いやすさ    | 詳細度                   |
| -------------------------- | ------------ | ------------------- | ------------------------ |
| **駅データ.jp**            | 高い         | ★★★★★ (CSV)         | 路線との紐付けが強い     |
| **公共交通オープンデータ** | リアルタイム | ★★★★☆ (JSON)        | 駅構内・出口情報まで豊富 |
| **国土数値情報**           | 年1回程度    | ★★★☆☆ (GeoJSON/GIS) | 地図描画・統計解析向き   |

### 信頼性の担保

- 画面下部に「Source: MLIT Real Estate Information Library」を常駐。
- 計算ロジック（重回帰の決定係数など）を数秒間、背景にテクスチャとして流すことでプロフェッショナリズムを演出。

## 6. 展開戦略と拡張性（Scalability）

- **完全自動化プログラム:** エリア名と期間を指定すれば、分析からレンダリングまで一貫して行うコードを構築。
- **多角展開:** 同じコードを用い、東京の次は大阪、福岡、札幌とエリアを広げ、シリーズとしての認知を確立する。
- **インタラクティブ化:** 反響のあったエリアについては、ユーザーが自由に視点を変えられるWebベースのダッシュボードへの誘導も視野に入れる。
