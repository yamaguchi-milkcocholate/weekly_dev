# Asset Image Gen PoC

## 目的

Tripo AIでの3Dモデル生成に使用するマルチビュー画像（front, back, side）を、Gemini 3.1 Flash Imageで生成する。

1枚の参照画像+テキスト説明から、同一オブジェクトの正面・横・後ろの画像を一貫性を保って生成する。

## 技術構成

- **モデル**: `gemini-3.1-flash-image-preview`（Nano Banana 2、高速・大量処理向け）
- **API**: httpx直接REST API呼び出し（3dcg_poc6と同パターン）
- **入力**: オブジェクト画像1枚 + テキスト説明 + ビュー別プロンプト
- **出力**: front.png, side.png, back.png
- **背景**: グレー(#808080)、影なし（Tripo背景除去に最適）

## 検証結果

### アプローチ比較

| アプローチ | 方式 | 結果 |
|---|---|---|
| A: 独立生成 | 各ビューを入力画像のみから独立リクエスト | **採用** |
| B: 連鎖参照 | 前ビューの出力を次のリクエストに参照画像として追加 | 不採用 |

- **アプローチB（連鎖参照）の問題**: sideビューがfrontに引きずられ、正面構図のまま生成される。前ステップの出力画像が強い参照になりすぎて、アングル変更が効かない
- **アプローチA（独立生成）を採用**: 各ビューが独立しているため、ビュー固有のプロンプトが正しく反映される

### Geminiの複数画像生成について

- 1リクエストで複数画像を生成するパラメータは存在しない
- 各ビューは個別リクエストで生成する必要がある

### プロンプト設計の知見

**対象ごとにビュープロンプトを変える必要がある。** 汎用的なビュー指示（「右90度から撮影」等）ではオブジェクトの種類によって構造が崩れる。

#### キッチン（壁付き複合オブジェクト）の場合

- **課題**: 汎用sideプロンプトでは正面のディテールに引きずられ、構造が崩壊する
- **解決策**: 具体的な寸法・見えるべき要素・見えないべき要素を明示
- sideプロンプト例: 「壁パネル断面15cm×200cm、キャビネット側面60cm×85cm。正面のコンロ・シンク・引き出しは見えない」
- backプロンプト例: 「壁裏面、配管、ダクト、電気配線を描画。正面ディテールは不可視」
- 結果: `output/kitchen_v5/` — front/side/back全て構造保持に成功

#### ソファ（独立家具）の場合

- 独立家具は各ビューの構造が明確なため、比較的簡単にうまくいく
- L字構成・オットマン位置・背面パネルなどオブジェクト固有の特徴を記述すれば十分
- 結果: `output/sofa_v1/` — 3ビュー全て良好。Tripo入力として十分なクオリティ

### 生成品質まとめ

| 対象 | front | side | back | 総合 |
|---|---|---|---|---|
| キッチン（壁なし・v1） | ○ | △ フレーミング偏り | △ front類似 | △ |
| キッチン（壁あり・v5） | ○ | ○ 真横90度 | ○ 配管リアル | ○ |
| ソファ（v1） | ○ | ○ 側面プロファイル正確 | ○ 背面パネル正確 | ○ |

## 実行方法

```bash
uv run python poc/asset_image_gen/generate_views.py \
  --image path/to/object.png \
  --description "オブジェクトの説明" \
  --front "frontビューの指示" \
  --side "sideビューの指示" \
  --back "backビューの指示" \
  --output-dir poc/asset_image_gen/output
```

## 結論

**API経由の生成品質はGemini Webアプリに大きく劣る。** Webアプリで手動生成した方がよほど良い結果が得られる。API経由ではプロンプトの工夫に限界があり、特にビュー変換の構造保持が不安定。自動パイプライン化のメリットよりも品質差が大きいため、現時点ではWebアプリでの手動生成を推奨する。

## 備考

- Tripo AIはマルチビュー入力（最大4画角）に対応
- 正面・側面・背面の3画角があれば3D再構成の品質が大幅に向上
- グレー背景で生成することで背景除去の手間を省く
- 生成時間: 1ビューあたり約20〜50秒
