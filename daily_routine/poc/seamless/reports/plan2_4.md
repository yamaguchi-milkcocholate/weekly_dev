## Phase 2: ポーズ変更（自撮り化）

### 目的

seed キャプチャの直立ポーズを、スマホ内カメラで自撮りしている構図に変更できるかを検証。

### 実験パターン

| パターン | 方式                                      | API            | コスト |
| -------- | ----------------------------------------- | -------------- | ------ |
| P-A      | 1パス（人物差し替え + ポーズ変更）        | Kontext Pro    | $0.04  |
| P-B      | 2パス（差し替え → ポーズ変更）            | Kontext Pro x2 | $0.08  |
| P-C      | 1パス（ポーズ変更のみ、人物差し替えなし） | Kontext Pro    | $0.04  |

### 結果

| パターン | ポーズ反映度                 | キャラクター一致度     | 環境再現度     | 自然さ |
| -------- | ---------------------------- | ---------------------- | -------------- | ------ |
| **P-A**  | ◎ 腕伸ばし自撮り             | ◎ 彩花の服装・髪型一致 | ◎ 地下通路保持 | ◎      |
| **P-B**  | △ スマホ持ちだが腕が伸びない | ◎ 彩花一致             | ◎ 地下通路保持 | ○      |
| **P-C**  | ◎ 自撮り構図                 | — (元の人物)           | ◎ 地下通路保持 | ◎      |

### 結論

**P-A（1パス）が最良**。Phase 1 と同様、1パスで複数変更を同時指示する方式が最も品質が高い。P-B の2パス目ではポーズ変更が弱く（腕が十分に伸びない）、Phase 1 で確認した連鎖劣化と同じ傾向。

## Phase 3: Runway / Gemini 比較

### 目的

Phase 2 で FLUX Kontext P-A が有効であることを確認したが、ポーズを大きく変更する場合は Kontext の構図保持の強みが活きていない。Runway（T2I）や Gemini（I2I）でも同等以上の結果が得られるか検証。

### 実験パターン

| パターン | 方式                        | API                | コスト |
| -------- | --------------------------- | ------------------ | ------ |
| R-A      | T2I + @char + @location     | Runway Gen-4 Image | $0.05  |
| R-B      | T2I + @char のみ            | Runway Gen-4 Image | $0.05  |
| G-A      | I2I テキスト指示のみ        | Gemini 3 Pro       | $0.04  |
| G-B      | I2I 参照画像 + テキスト指示 | Gemini 3 Pro       | $0.04  |

### 結果

| パターン | ポーズ反映度         | キャラクター一致度     | 環境再現度                                 | 自然さ |
| -------- | -------------------- | ---------------------- | ------------------------------------------ | ------ |
| **R-A**  | ◎ 腕伸ばし自撮り     | ◎ 服装・髪型一致       | ○ 地下通路風（新規構図）                   | ◎      |
| **R-B**  | △ 腕が伸びず全身引き | ◎ 服装・髪型一致       | ○ 地下通路風                               | ○      |
| **G-A**  | ◎ 腕伸ばし自撮り     | ◎ 全衣装ディテール一致 | ◎ 地下通路の柱・タイル・案内板を忠実に保持 | ◎      |
| **G-B**  | ◎ 自撮り構図         | ◎ 服装・髪型一致       | ◎ seed 構図保持                            | ◎      |

### 生成画像

| R-A                              | R-B                              | G-A                              | G-B                              |
| -------------------------------- | -------------------------------- | -------------------------------- | -------------------------------- |
| `generated/plan3/R-A/output.png` | `generated/plan3/R-B/output.png` | `generated/plan3/G-A/output.png` | `generated/plan3/G-B/output.png` |

### 結論

**Gemini 3 Pro（G-A/G-B）が最良**。G-A はテキスト指示のみで、キャラクター一致・自撮りポーズ・背景保持の全てが高品質。Runway R-A も高品質だが、T2I のため背景は新規生成であり seed 構図の保持度では Gemini の I2I に劣る。

**FLUX Kontext P-A と Gemini G-A が同等品質で、両方 $0.04**。

## Phase 4: 環境変更（背景差し替え）

### 目的

人物をそのままに、背景環境だけを桜の川沿い遊歩道に変更する。Kontext の「構図を保持しつつ部分的に編集する」強みが最も活きるケースで、Gemini との品質差を検証。

### 実験パターン

| パターン | 方式                      | API          | コスト |
| -------- | ------------------------- | ------------ | ------ |
| E-A      | テキストのみで背景変更    | Kontext Pro  | $0.04  |
| E-B      | sakura.jpg 参照で環境転写 | Kontext Max  | $0.08  |
| E-C      | テキストのみで背景変更    | Gemini 3 Pro | $0.04  |
| E-D      | sakura.jpg 参照で環境転写 | Gemini 3 Pro | $0.04  |

### 結果

| パターン | 環境反映度                       | 人物保持度                       | 合成の自然さ                 | 参照忠実度 |
| -------- | -------------------------------- | -------------------------------- | ---------------------------- | ---------- |
| **E-A**  | ◎ 桜並木+川沿い                  | ◎ ポーズ・服装・イヤホン保持     | ○ テキストオーバーレイ残存   | —          |
| **E-B**  | ◎ sakura.jpg の構図を忠実に転写  | ◎ 服装・イヤホン保持             | ○ アスペクト比が正方形に変化 | ◎          |
| **E-C**  | ◎ 桜の川沿い遊歩道、非常にリアル | ◎ ポーズ・服装・イヤホン完全保持 | ◎ 照明が春の自然光に自然変化 | —          |
| **E-D**  | ○ 桜+川+建物あり                 | △ 人物にやや合成感               | △ 人物と背景の境界が不自然   | ○          |

### 生成画像

| E-A                              | E-B                              | E-C                              | E-D                              |
| -------------------------------- | -------------------------------- | -------------------------------- | -------------------------------- |
| `generated/plan4/E-A/output.png` | `generated/plan4/E-B/output.png` | `generated/plan4/E-C/output.png` | `generated/plan4/E-D/output.png` |

### 結論

**E-C（Gemini テキストのみ）が最良**。背景の桜並木が最もリアルで、照明も地下通路の蛍光灯から春の自然光に自然に変化。人物の保持度も最高。

Kontext の構図保持が最も活きるはずの「背景のみ変更」ケースでも、Gemini 3 Pro が同等以上の品質を示した。

参照画像を追加するとかえって品質が下がる傾向は Kontext（Phase 1 の D-A）でも Gemini（E-D）でも共通しており、テキスト指示のみの方が安定する。

## 総合結論

### API 比較サマリ

| API                | ポーズ変更 | 人物差し替え | 環境変更     | コスト | 総合                 |
| ------------------ | ---------- | ------------ | ------------ | ------ | -------------------- |
| **Gemini 3 Pro**   | ◎          | ◎            | ◎            | $0.04  | **最良**             |
| FLUX Kontext Pro   | ◎          | ◎            | ○            | $0.04  | 良好                 |
| FLUX Kontext Max   | —          | ○            | ◎ (参照忠実) | $0.08  | 高コスト             |
| Runway Gen-4 Image | ◎          | ◎            | — (T2I)      | $0.05  | 良好（背景保持不可） |

### 確立された知見

1. **1パスが最良**: 人物差し替え・ポーズ変更・環境変更のいずれも、1パスで同時指示する方式が最も品質が高い（Phase 1-2 で確認）
2. **連鎖（マルチパス）は劣化する**: パスを重ねるほど画質が劣化し、意図した調整効果は薄い
3. **参照画像の追加は逆効果になりうる**: テキスト指示のみの方が安定した結果を出す傾向（Kontext D-A, Gemini E-D）
4. **Gemini 3 Pro は全タスクで Kontext と同等以上**: ポーズ変更・人物差し替え・環境変更の全てで高品質。$0.04 で最もコストパフォーマンスが高い

### 所感・次のステップ

**Gemini 3 Pro を Core として採用したい**。全タスクで安定した品質を示し、コストも最安。

ただし、以下の点で追加調査が必要:

- **Gemini に特化したプロンプトチューニング手法**: 現在のプロンプトは Kontext 向けに設計されたもの。Gemini の I2I 特性に最適化したプロンプト設計でさらに品質向上の余地がある
- **環境入れ替えのコツ**: E-D で参照画像を入れると品質が下がった。Gemini における複数画像入力時の役割指定方法や、環境転写のベストプラクティスを調査する
- **テキストオーバーレイの除去**: seed 画像のテキスト（7:40 等）が残存する問題への対処

次回、Gemini 特化の調査・チューニングを実施する。
