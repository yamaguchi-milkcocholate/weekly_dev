# Phase A-5: テキスト/オーバーレイ除去 — 検証レポート

## 実験概要

| 項目       | 値                                                        |
| ---------- | --------------------------------------------------------- |
| 目的       | seed 画像上の時刻表示・テロップを自然に除去する手法の確立 |
| モデル     | `gemini-3-pro-image-preview`                              |
| SDK        | `google-genai` (Python SDK)                               |
| 入力構成   | seed 画像 1 枚 + テキスト指示（参照画像なし）             |
| パターン数 | 4                                                         |
| Seed 画像  | 3 枚（デスク・鏡・カフェ）                                |
| 生成数     | 12（成功 8 / 失敗 4）                                     |
| 実コスト   | $0.32                                                     |

## Seed 画像のテキストオーバーレイ

| Seed | シーン         | オーバーレイ内容                                              |
| ---- | -------------- | ------------------------------------------------------------- |
| 1    | デスク・室内   | 「4:49 起床する」（中央、短テロップ）                         |
| 4    | 鏡・玄関       | 「6:40 出発」（中央、短テロップ）                             |
| 8    | カフェ・スタバ | 「7:59 モチベ上げるために定期的にスタバ」（中央、長テロップ） |

## 検証パターン

| ID  | 名前                     | 方針                                                |
| --- | ------------------------ | --------------------------------------------------- |
| T1  | 簡潔指示（ベースライン） | `Remove all text overlays from this image.`         |
| T2  | 除去対象を具体的に指定   | timestamp + Japanese text を明示 + Reconstruct 指示 |
| T3  | ALL CAPS 保持指示付き    | 除去指示 + MUST/MUST NOT で人物・背景保持を明示     |
| T4  | 編集モード指示（Clean）  | コンテキスト説明 + `Clean` 動詞で編集モード誘導     |

## 結果

### パターン別評価

| パターン | seed_1 (デスク)            | seed_4 (鏡)            | seed_8 (カフェ)            | 成功率 |
| -------- | -------------------------- | ---------------------- | -------------------------- | ------ |
| T1       | ◎ 完全除去、背景復元良好   | ◎ 完璧な保持           | ◎ テキスト除去、目隠し保持 | 3/3    |
| T2       | ✗ 生成失敗（フィルタ）     | ✗ 生成失敗（フィルタ） | ✗ 生成失敗（フィルタ）     | 0/3    |
| T3       | ✗ 生成失敗（フィルタ）     | ◎ T1 と同等品質        | ◎ T1 と同等品質            | 2/3    |
| T4       | ◎ 右下に小アイコン残存あり | ◎ T1 と同等品質        | ◎ T1 と同等品質            | 3/3    |

### 評価基準

- **テキスト除去の完全性**: オーバーレイが完全に除去されているか
- **背景復元の自然さ**: テキストがあった箇所の復元品質
- **人物保持**: 人物の外見・ポーズ・服装が変化していないか
- **構図保持**: 画角・構図が元画像と一致しているか

### 詳細評価

#### T1: 簡潔指示（ベースライン）— **推奨**

- 全 3 seed で成功。テキストオーバーレイが完全に除去された
- 背景復元品質は非常に高く、テキストがあった箇所も自然
- 人物・構図・照明すべて高精度に保持
- **最もシンプルで最も安定**

#### T2: 除去対象を具体的に指定 — **不可（全滅）**

- 全 3 seed で `parts=None`（レスポンスにコンテンツなし）
- API は 200 OK を返すが画像が生成されない → セーフティフィルタの可能性
- 「Japanese text」「Reconstruct」のいずれかがトリガーと推測

#### T3: ALL CAPS 保持指示付き — **不安定**

- seed_4, seed_8 は成功し T1 と同等品質
- seed_1 で失敗。MUST/MUST NOT の保持指示がプロンプト全体を重くし、フィルタに引っかかった可能性
- 成功時の品質に T1 との差は見られない → **追加の保持指示は不要**

#### T4: 編集モード指示（Clean）— **良好だが T1 で十分**

- 全 3 seed で成功。テキスト除去品質は T1 と同等
- seed_1 で右下に小さなアイコンが追加される副作用あり
- コンテキスト説明が過剰で、プロンプトの複雑さに見合うメリットなし

## 主要な発見

### 1. シンプルなプロンプトが最強

テキスト除去は Gemini にとって比較的容易なタスク。`Remove all text overlays from this image.` の 1 文で十分。追加の保持指示・コンテキスト説明はむしろ成功率を下げる。

### 2. 具体的なテキスト内容の記述は逆効果

T2 のように除去対象を具体的に記述すると、セーフティフィルタに引っかかる。特に「Japanese text」という表現は避けるべき。

### 3. 保持指示は不要

他フェーズ（A-1, A-3）では有効だった ALL CAPS 保持指示が、テキスト除去では不要。テキスト除去は変更範囲が限定的なため、モデルは指示がなくても人物・背景を正確に保持する。

### 4. 顔匿名化マスクは保持される

seed 画像に含まれるハート型の顔隠し（seed_1）や横棒の目隠し（seed_8）は「テキスト」として除去されず、画像の一部として正しく保持される。パイプラインの前処理として安全に使用可能。

### 5. 背景復元品質は高い

テキストが人物や複雑な背景（カフェの壁・額縁など）に重なっていても、自然に復元される。Gemini の inpainting 能力は実用レベル。

## 横断的知見（他フェーズへの影響）

- **テキスト除去は前処理として十分実用的**: B4（複合編集）の前段で使用可能
- **プロンプトは短いほど安定**: テキスト除去に限らず、変更範囲が限定的なタスクでは詳細な指示が逆効果になりうる
- **セーフティフィルタの回避**: 除去対象の内容（言語・テキスト内容）を具体的に記述しないこと
