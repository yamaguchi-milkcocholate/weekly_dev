# 画像・動画生成プロンプトのレビュースキル

画像生成プロンプトおよび動画生成（I2V）プロンプトを作成・レビューする。

## 前提知識

このスキルを実行する前に、以下のファイルを必ず読み込むこと:

1. **プロンプトガイドライン**: [/docs/guidelines/visual_prompt.md](/docs/guidelines/visual_prompt.md) — 画像・動画生成プロンプトのベストプラクティス
2. **ADR-003**: [/docs/adrs/003_video_generation_workflow.md](/docs/adrs/003_video_generation_workflow.md) — 採用済みの動画生成ワークフローと技術的制約

## 入力

$ARGUMENTS

入力は以下のいずれかの形式:

- プロンプトテキスト（直接記述）
- プロンプトを含むファイルパス（`.json`, `.yaml`, `.py` 等）
- 入力なし（対話的にプロンプト作成を支援）

## 実行手順

### プロンプトのレビュー時

1. ガイドラインを読み込む
2. 入力されたプロンプトを分析する
3. ガイドラインのチェックリストに照合し、問題点を報告する
4. 改善版のプロンプトを提案する

### プロンプトの作成時

1. ガイドラインを読み込む
2. ユーザーからシーンの意図（場所、状況、動き）をヒアリングする
3. ガイドラインに準拠したプロンプトを作成する
4. チェックリストで自己検証してから提示する

## レビュー観点

### 画像生成プロンプト

ガイドライン「5. チェックリスト > 画像生成プロンプト」の各項目を検証する:

- 英語で記述されているか
- 完全な文章になっているか（キーワードリストでないか）
- 場所・環境が明記されているか
- 1シーンのみを記述しているか
- 照明・雰囲気が含まれているか

### 動画生成プロンプト

ガイドライン「5. チェックリスト > 動画生成プロンプト」の各項目を検証する:

- 英語で記述されているか
- Subject Motion（被写体の動き）が含まれているか
- Scene Motion（環境の動き）が含まれているか
- Camera Motion（カメラワーク）が含まれているか
- 入力画像に既にある情報を再記述していないか
- 能動態の精密な動詞を使用しているか
- 否定表現を使っていないか
- 1ショットにつき1つの連続した動きのみか

### 動詞の品質

- 汎用的な動詞（moves, does, goes）を使っていないか
- ガイドライン「3. 動詞リファレンス」を参照し、精密な動詞への置き換えを提案する

## 出力形式

レビュー結果は以下の形式で報告する:

1. **判定**: OK / 要改善
2. **チェックリスト結果**: 各項目の合否
3. **問題点**（ある場合）: 具体的な箇所と理由
4. **改善案**（ある場合）: 修正後のプロンプト全文
