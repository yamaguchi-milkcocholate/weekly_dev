# ビジョンと戦略

## 目的

家族・大切な人・自分に使える時間を増やすために、場所・時間に縛られない働き方を実現する。

## 事業の軸

**「リアル制約 × AI創造」の掛け算。**

- リアル空間の制約を忠実にデジタル化した上で、AIがその制約の中で創造・予測する
- 忠実な再現だけなら測量・CADと変わらない。AIの創造だけなら現実の制約を無視した絵空事になる
- **掛け合わせることで「現実に実装可能なアイデアが自動で生まれる」という唯一無二の価値を作る**

## 戦略

「デジタルだけで完結するスキルはAIに代替される」という認識のもと、**非構造データ（画像・3D・動画等）を扱うAIパイプラインを、業界横断で組み立てられる人材**になる。

データサイエンティストの経験を土台に、非構造データ × AI APIのパイプライン構築力を武器とする。特定業界に深入りするのではなく、複数領域を横断する。

## 事業モデル: 段階的移行

**Phase 1（現在）: パイプライン構築型** — 副業として開始
- 業界側がデータを持っている → 自分がAIパイプラインを組んで価値に変える
- データサイエンティストの延長で案件を取れる
- 複数領域を横断しやすい

**Phase 2（将来）: フルスタック型** — 興味を持てる領域が見つかれば移行
- データ取得から成果物納品まで一気通貫でやる
- パイプライン自体が競争優位になる（外から見えない）
- Phase 1で複数領域に関わる中で、自然と深入りしたくなる領域が見つかれば移行する。見つからなければPhase 1のまま続ける

## スコープ

アウトプットは業界の課題に応じて変わる。映像・可視化などのデジタル成果物に限らず、製造・加工などのフィジカル成果物も含む。

```
リアル空間 → データ取得 → AIパイプライン → アウトプット → リアル空間に反映
```

有望な応用領域:
- **フィジカルAI**: リアル空間を忠実に3D化 → ロボット・自動運転等がリアル空間で活動するのをスムーズにする
- **空間デザイン**: 間取り+レイアウトのシミュレーション・可視化（最初の実績作り）
- その他、業界ごとにデータ取得方法が異なるため独自パイプラインが必要な領域

## 目指す姿

**民間・個人〜中小企業レベルの「リアルデータ統合可視化・予測システム」を構築できる人材。**

パランティアのMaven Smart System（リアルデータ統合 → AI分析 → 意思決定支援）と構造は同じ。ただしスケールが異なる。パランティアは国家・大企業しか相手にしないが、同じニーズは民間にもある。

```
パランティア: 国家・軍事レベルのリアルデータ → AI → 意思決定支援
山口:         民間・中小レベルのリアルデータ → AI → 意思決定支援
```

## 核心の価値

- 非構造データ（画像・3D・動画）を複数のAI APIで変換する一気通貫パイプラインを構築できる
- 業界ごとにデータの形式・取得方法が異なるため、独自パイプラインの構築が必要になる
- データサイエンス + AIパイプライン構築の両方がわかる人材がそこに必要

## 現在のアプローチ

- PoC 0-9で「間取り図→3Dウォークスルー動画」の一気通貫パイプラインを構築済み
- これは非構造データパイプライン構築力の証明であり、一例として空間データを扱ったもの
- パイプラインの入口（データ形式・取得方法）を差し替えることで他領域にも対応できる

## 成長パス

```
✅ パイプラインを作る（技術力の証明） ← PoC 0-9で完了
→ 発信（技術記事 + 簡素な動画）+ 受託開発で実績とお金を積む
→ 複数領域に関わる中で、自分が没頭できる事業テーマを探す
→ 見つかればフルスタック型に移行 → 独立
```

**注意: 受託開発は手段であり目標ではない。** 受託で安定することに満足せず、常に「自分自身の事業」を探し続けること。受託は実績・資金・業界解像度を得るための期間限定の手段。
