# PoC Step 2: 画像から挿入可能なアセットを3D空間データとして取得できる

> [ロードマップに戻る](./roadmap.md)

## 検証目標

画像を入力として、レイアウトに必要な家具をAIが分類・抽出し、ユーザーが選択したオブジェクトを3D空間に挿入可能なアセットとして取得できる。

---

## アセットとは

3D空間に挿入可能な個別オブジェクトの3Dデータ。

```
アセット
└── 形状データ: オブジェクトの3D座標・形状
```

テクスチャ（素材・色・質感）はアセットには含めない。後工程でユーザーのイメージ画像からAIが適用する。

---

## オブジェクト選定のフロー

```
画像入力
    ↓
AIがカテゴリ分類（家具など）で候補を抽出
    ↓
ユーザーが候補から使うものを選択
    ↓
選択されたオブジェクトのみを3D化
    ↓
不足分は個別アセットで追加
```

---

## 入力画像の種類と難易度

```
パターンA: インテリアECサイトの商品画像
└── オブジェクトが単体・白背景で写っている
└── 切り出し不要、3D化に集中できる
└── 難易度: 低

パターンB: 別部屋の写真
└── 複数の家具・小物が混在している
└── オブジェクトの認識・切り出しが必要
└── 難易度: 高
```

PoCではパターンAから検証し、成功後にパターンBへ進む。

---

## 検証ステップ

### Step 1: ECサイト商品画像からアセットを取得できるか（パターンA）

- [ ] インテリアECサイトからサンプル商品画像を収集する（ソファ・テーブル・椅子など）
- [ ] 商品画像（単体・白背景）からオブジェクトの3Dデータを生成できる
- [ ] 生成した3Dデータを目視で確認する
  - オブジェクトの形状が元の画像と概ね一致しているか
- [ ] 3D空間に挿入可能な形式で出力できる

### Step 2: 別部屋の写真からアセットを取得できるか（パターンB）

- [ ] 「いいな」と思う部屋の写真を用意する
- [ ] AIが画像内のオブジェクトを家具カテゴリで正しく認識・分類できる
  - レイアウトに必要な家具（ソファ・テーブル・棚・照明など）が抽出されるか
  - コップ・リモコンなどの不要なオブジェクトが除外されるか
- [ ] ユーザーが選択したオブジェクトのみを切り出して3D化できる
- [ ] 生成した3Dデータを目視で確認する
  - Step 1と比較して形状の精度はどうか
- [ ] 3D空間に挿入可能な形式で出力できる

---

## 合格基準

**この段階では品質よりも「アセットとして成立しているか」を確認することを優先する。**

| 評価項目             | 合格の目安                                           |
| -------------------- | ---------------------------------------------------- |
| カテゴリ分類の妥当性 | 家具カテゴリのオブジェクトが概ね正しく抽出されている |
| 形状の妥当性         | オブジェクトの形状が元の画像と概ね一致している       |
| 次ステップへの接続   | 3D空間への挿入が可能な形式で出力できる               |

- パターンA・Bそれぞれのアセット出力結果

- カテゴリ分類の精度と課題の考察
- 次のステップ（AIによるレイアウト提案・3D空間への挿入）への接続方針
