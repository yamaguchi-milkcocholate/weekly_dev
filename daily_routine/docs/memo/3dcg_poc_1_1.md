# PoC Step 1: 技術方針（改訂版）

## 基本方針

Blenderで間取り図を背景に読み込み、壁を手動トレースして3D化する。
GUIは自作しない。OSSの自動認識も使わない。Blenderの既製機能を使い倒す。

```
Blenderで完結する作業
└── 間取り図画像を背景として読み込む
└── 壁をトレースして2D形状を作成する
└── 押し出し（Extrude）で3D化する
└── ドア・窓の開口部を切り抜く
└── 任意視点からレンダリングする
```

---

## パイプライン

```
【Step 1】Blenderで間取り図を背景に読み込む
    ↓
【Step 2】壁の輪郭をトレース（2D）
    └── 間取り図に合わせて端点をクリックして壁を描く
    └── ドア・窓の位置も指定する
    ↓
【Step 3】押し出し（Extrude）で3D化
    └── 2D形状を高さ方向に押し出して壁を生成する
    └── 床・天井も生成する
    └── ドア・窓の開口部を切り抜く
    └── 出力: テクスチャなし3Dメッシュ（骨格）
    ↓
【Step 4】部屋写真 + 撮影位置で細部を補強（MASt3R）
    └── 人が部屋写真を撮影し、撮影位置（x, y, 向き）を指定する
    └── MASt3Rが写真から細部・テクスチャを補強する
    └── 出力: テクスチャ付き3Dメッシュ
    ↓
【Step 5】Blenderで任意視点からレンダリング
    └── 3D空間上でカメラを操作して視点を決める
    └── Cycles / EEVEEでレンダリングする
```

---

## Blenderを選ぶ理由

```
GUIを自作する場合
└── 壁の端点入力・編集機能を一から実装
└── UXの作り込みに時間がかかる
└── スキルがパイプライン外に流れる

Blenderを使う場合
└── 壁を描く・端点を指定する機能が既にある
└── 3D化・レンダリングも同じツールで完結
└── Blenderスキルはパイプライン全体に使える
└── 「現実データ × デジタル」の核心スキルと直結する
```

---

## 学習ステップ

Blenderの基本操作を以下の順で習得する。

```
1. 背景画像の読み込み（Image as Planes / Background）
2. 頂点・辺の操作（Edit Mode）
3. 押し出し（Extrude Region）
4. ブーリアン演算（開口部の切り抜き）
5. カメラ設定とレンダリング（Cycles / EEVEE）
```

## これまでの検証結果

| アプローチ                 | 結果        | 理由                                                     |
| -------------------------- | ----------- | -------------------------------------------------------- |
| Gemini API                 | ❌ 不適     | 日本の間取り図の内壁・ドア位置の認識精度が実用水準に遠い |
| 手動GUI自作                | ❌ 中断     | 実装コストが高くスキルがパイプライン外に流れる           |
| CubiCasa5k / DeepFloorplan | ❌ 採用せず | 出力がセグメンテーションマップであり手動作成と大差ない   |
| **Blender手動モデリング**  | ✅ 採用     | 既製GUIで確実・高精度。スキルがパイプライン全体に活きる  |
