# 間取り可視化パイプライン ロードマップ

## 背景と目的

### 目的

家族・大切な人・自分に使える時間を増やすために、場所・時間に縛られない働き方を実現する。

### 戦略

「デジタルだけで完結するスキルはAIに代替される」という認識のもと、**現実空間のデータをデジタルに変換する技術**を核心スキルとして積み上げる。動画クリエイターと同じ土俵では戦わず、「現実データ × AI映像生成」という独自領域を開拓する。

### 最初のプロジェクト: 間取り可視化ツール

- **Why**: 多くの人が悩む課題（部屋のレイアウト）を扱い、SNSでの共感・拡散が見込める
- **Why now**: 自分の部屋ですぐに始められる。既存AIツールでは精度が出ない領域
- **最終形**: ツールを作り、そのツールで動画コンテンツを作り、スキルを広く知ってもらう

---

## ゴール設定

```
ツールを作る（技術力の証明）
    ↓
ツールで動画を作る（わかりやすい実績）
    ↓
動画をSNSで発信（不動産・インテリア業者の目に触れる）
    ↓
クライアントから声がかかる（現場データへのアクセス権を得る）
    ↓
より高度なパイプラインに発展（自動車部品・製造現場など）
```

---

## パイプライン設計（実現したいこと）

```
Input: 間取り図（画像 or PDF）+ 部屋写真 + ユーザーのイメージ画像
    ↓
3D空間データの構成
    └── 空間の形状を把握できる（壁・床・天井の座標）
    ↓
アセットの取得と配置
    └── 部屋写真・商品画像から家具などのアセットを3Dデータとして取得できる
    └── AIが空間の広さ・形状を根拠にレイアウトを提案できる（配置座標の出力）
    └── 提案されたレイアウトで3D空間にアセットを挿入できる
    └── ユーザーが手動でアセットの位置・向きを調整できる
    ↓
任意視点の設定
    └── 3D空間上でカメラを操作して視点を指定できる
    └── 指定した視点からレンダリングできる
    ↓
リアルな映像の生成
    └── 3Dレンダリング結果 × ユーザーのイメージ画像から
        テクスチャ・雰囲気が反映されたリアルな部屋映像を生成できる
    ↓
Output: ユーザーのイメージが反映された任意視点のリアルな部屋映像
```

## ロードマップ

### Phase 0: 技術検証（PoC）

**目標**: パイプラインの各ステップが実現可能かを確認する

- [ ] 間取り図と部屋写真から3D空間データを構成できる → [詳細](./3dcg_poc_1.md)
- [ ] 画像から挿入可能なアセットを3D空間データとして取得できる → [詳細](./3dcg_poc_2.md)
- [ ] AIが空間とアセットを元にレイアウトを提案できる → [詳細](./3dcg_poc_3.md)
- [ ] 提案されたレイアウトで3D空間にアセットを挿入できる → [詳細](./3dcg_poc_4.md)
- [ ] 構成した3D空間データを任意のカメラ位置からレンダリングできる → [詳細](./3dcg_poc_5.md)
- [ ] 3Dレンダリング結果とユーザーのイメージ画像からテクスチャ・雰囲気が反映されたリアルな部屋映像を生成できる → [詳細](./3dcg_poc_6.md)
- [ ] 上記をエンドツーエンドで一度通せる

**成果物**: 動くプロトタイプ（品質は問わない）

---

### Phase 1: 最小機能のツール構築

**目標**: 間取り図を入力すると任意視点の部屋映像が出力されるツール

- [ ] 間取り図から壁・部屋・窓を認識できる
- [ ] カメラ位置を指定して映像を生成できる
- [ ] 複数のレイアウト案を一括生成できる
- [ ] 自分が実用的に使える状態になっている

**成果物**: ローカルで動くツール

---

### Phase 2: デモコンテンツの制作

**目標**: ツールを使った動画コンテンツを作り、SNSで発信する

- [ ] 「間取り図 → 複数レイアウト映像」のデモ動画を制作できる
- [ ] SNSに公開し、初期の反応を得られる
- [ ] フィードバックをツール改善に反映できる

**成果物**: 公開された動画コンテンツ

---

### Phase 3: クライアント獲得・領域拡張

**目標**: 特定領域のクライアントと協働し、独自データ資産を構築する

- [ ] 不動産・インテリア業者と協働できる
- [ ] クライアントの現場データを使ったパイプラインを構築できる
- [ ] 現場での実データ収集からの映像生成が実現できる
- [ ] 「複雑な現場を誰でも理解できる映像にする」応用領域に展開できる
  - 自動車部品の構造可視化
  - 製造現場の工程可視化

**成果物**: 有償クライアントワーク、独自データ資産

## 現在の課題との接続

本ロードマップは、既存の動画生成パイプライン（`storyboard → environment → keyframe`）で発生している以下の課題を、技術的に根本解決するものでもある。

| 既存課題                                    | 本ロードマップでの解決方向                                                                                   |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| camera_workと環境画像の画角不整合（Doc 1）  | 3D空間データから任意のカメラ位置でレンダリングすることで、カット単位で正確な画角の環境情報を渡せるようにする |
| テキスト指示による画角変更が不安定（Doc 2） | テキストではなく3D空間データで画像生成を制御する                                                             |
