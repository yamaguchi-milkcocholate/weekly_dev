# PoC Step 3: AIが空間とアセットを元にレイアウトを提案できる

> [ロードマップに戻る](./roadmap.md)

## 検証目標

3D空間データとアセットを入力として、AIが空間の広さ・形状を根拠に各アセットの配置座標を提案できる。

---

## このステップの役割

```
空間の広さ・形状を根拠に
└── 物理的に配置可能か（アセットが空間に収まるか）
└── 動線が確保されているか
を判断した配置座標を出力する
```

テクスチャ・雰囲気はこのステップでは扱わない。後工程「3Dレンダリング結果 × ユーザーのイメージ画像」で適用する。

---

## 出力形式

各アセットの3D空間内の配置座標（位置・向き）。次のステップ「3D空間へのアセット挿入」への直接入力となる。

---

## 検証ステップ

### Step 1: 空間の広さ・形状を根拠にレイアウトを提案できるか

- [ ] Step 1（3D空間データ）・Step 2（アセット）の出力を入力として与える
- [ ] AIが空間の広さ・形状に基づいて各アセットの配置座標を出力できる
- [ ] 出力された座標を目視で確認する
  - アセットが空間内に物理的に収まっているか
  - 動線が概ね確保されているか

---

## 合格基準

配置された結果を目視して概ね妥当であること。

| 評価項目           | 合格の目安                                             |
| ------------------ | ------------------------------------------------------ |
| 出力の成立         | 各アセットの配置座標が出力できる                       |
| 物理的妥当性       | アセットが空間内に収まり、動線が概ね確保されている     |
| 次ステップへの接続 | 出力座標を次ステップ（3D空間への挿入）にそのまま渡せる |

---

## 成果物

- 配置座標の出力結果
- 物理的妥当性の考察（収まらなかったケース・動線の問題など）
- 次のステップ（提案されたレイアウトで3D空間にアセットを挿入）への接続方針
