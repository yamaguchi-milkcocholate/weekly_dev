# PoC 7: ウォークスルー動画生成

作成日: 2026-03-30

## 目的

スタイル適用済みの複数カメラ画像間を、AI動画生成サービスで補間し、リアルなウォークスルー動画を作成する。

## 背景

- PoC 6でマルチアングルのスタイル一貫性のある静止画生成が完了
- 完成品映像としてリアルなウォークスルーが必要（静止画スライドショーでは不十分）
- 全フレームにスタイル適用するアプローチはコスト($40/動画)とフレーム間一貫性の問題がある

## 検証する2方式

### 方式B（優先）: Blenderウォークスルー動画 → AI動画スタイル転写（Video-to-Video）

```
Blenderでカメラパス動画をレンダリング（テクスチャ・小物・窓景色を作り込み済み）
→ 5秒程度のセグメントに分割
→ AI動画生成サービスのVideo-to-Video機能でセグメントごとにスタイル転写
→ セグメントを結合してウォークスルー動画完成
```

- **カメラ移動・構造はBlenderが完全に担保**（AIが構造を想像する必要がない）
- 壁・家具の歪み問題が原理的に発生しない
- **ちらつき対策**: 1セグメント=1回のAI生成（5秒程度）に収めることで、セグメント内の時間的整合性はモデルが保証
- **セグメント間の一貫性**: 3DCGを作り込んでAIの補完余地を最小化（re-skinningのみに限定）。入力の3DCGが一貫しているから出力も一貫する
- PoC6で得た「AIに追加させない」原則を動画にも適用

### 方式A（副次）: スタイル適用済み静止画間のAI補間

```
カメラ1（スタイル適用済み） → AI動画生成 → カメラ2（スタイル適用済み）
→ セグメントを結合してウォークスルー動画完成
```

- 始点・終点が同一パイプライン（Gemini）で生成済み → スタイル一貫性は担保
- AIは「間を埋める」だけ → ゼロからの動画生成より構造が安定する見込み
- **リスク**: カメラ間距離が大きいとAIが構造を想像する必要があり歪みが出る可能性（方式Bにはないリスク）

### 不採用とした方式

| 方式 | 問題 |
|------|------|
| AI動画生成（1枚→動画） | 見えない部分を想像する必要があり構造崩壊しやすい |
| Blender全フレーム→Geminiスタイル適用 | コスト高($40/動画)、フレーム間ちらつき |
| Blender素レンダリング動画のまま | リアル感がない。制作過程の素材としてなら使える |

## 検証対象サービス

- **Runway Gen-4**: Image-to-Video、First/Last Frame指定
- **Kling**: Image-to-Video
- **LUMA Ray**: Image-to-Video

## 検証ポイント

- カメラ移動時の壁・家具の構造維持
- カメラ間距離と破綻の関係（近い→安定、遠い→破綻の仮説）
- セグメント間のつなぎ目の自然さ
- 生成動画のfps・尺・解像度

## フェーズ

### Phase 1: 方式Bの検証（優先）
- Blenderでカメラ1→カメラ2の5秒ウォークスルー動画をレンダリング
- Runway/Kling/LUMA等のV2V機能でスタイル転写を比較
- 評価基準: 構造維持、スタイル反映、フレーム間ちらつき、動きの滑らかさ

### Phase 1b: 方式Aの検証（方式Bが不十分な場合）
- カメラ1→カメラ2のスタイル適用済み画像でRunway/Kling/LUMAの画像間補間を比較

### Phase 2: 全セグメント生成・結合
- 選定方式で全カメラ間のセグメントを生成
- セグメント結合、BGM追加で完成動画化

## 入力素材

- `poc/3dcg_poc6/output/カメラ1.png` 〜 `カメラ6.png`（スタイル適用済み、6アングル）

3. 結果が不十分なら方式A（静止画間補間）もフォールバックとして検証
