# Meshy vs Tripo 比較（2026年4月時点）

## 1. 機能面

| 項目 | Meshy | Tripo |
|------|-------|-------|
| **入力形式** | テキスト、単一画像、マルチ画像 | テキスト、単一画像、マルチ画像 |
| **出力形式** | GLB, STL, 3MF | GLB のみ |
| **最新モデル** | Meshy-6 | Tripo v3.0 |
| **最大ポリゴン数** | ~300K | ~2M |
| **生成速度** | 30-90秒 | 25-100秒（v2.5は25-30秒、v3.0は~100秒） |
| **テクスチャ** | PBRマテリアル対応 | PBR対応 |
| **リトポロジー** | Remesh機能あり（5クレジット） | Smart Low Poly機能あり |
| **リギング** | Auto-Rigging（5クレジット） | 自動リギング・自動アニメーション |
| **アニメーション** | 対応（3クレジット） | 自動アニメーション対応 |
| **スタイル変換** | なし | LEGO, Voxel, Voronoi, Minecraft, Cartoon, Clay |
| **プラグイン** | Blender, Unity, Unreal Engine | なし（API中心） |
| **セキュリティ認証** | SOC2 + ISO 27001 | なし |
| **3Dプリント** | 97%スライサー通過率、Multi-Color対応 | 対応（クリーントポロジー） |

### 機能面の所感

- **Tripo** はスタイル変換・自動アニメーション・高ポリゴン（2M）が強み
- **Meshy** はゲームエンジン統合・出力形式の多さ・エンタープライズ対応が強み

## 3. APIとサブスクどちらがコスト効率が良いか

### Tripo

- API: $0.01/クレジット = 40クレジット/モデル = **$0.40/モデル**
- サブスク（Professional年払い）: $11.94/月 ÷ 75モデル = **~$0.16/モデル**
- **結論: サブスクが圧倒的に安い**。月30モデル以上生成するならサブスク一択

### Meshy

- API: クレジット単価は非公開（Pro以上のサブスク内クレジットで利用）
- サブスク（Pro年払い）: $10/月 ÷ 33モデル = **~$0.30/モデル**
- **結論: サブスクが安い**。API単独課金の詳細は非公開のため、まずProプラン加入が前提

## 4. 総合判断

| ユースケース | おすすめ |
|-------------|---------|
| **コスト重視・大量生成** | **Tripo** — 同価格帯で2倍以上のモデル生成可能 |
| **ゲームエンジン統合** | **Meshy** — Unity/Unreal プラグインあり |
| **スタイル多様性** | **Tripo** — LEGO/Voxel等のスタイル変換が独自 |
| **出力形式の多様性** | **Meshy** — GLB/STL/3MF対応 |
| **エンタープライズ** | **Meshy** — SOC2/ISO認証、SAML SSO |
| **自動リギング・アニメーション** | **Tripo** — ビルトインで自動対応 |
| **3Dプリント** | **Meshy** — 高い印刷適性 |

## 5. 本プロジェクトでの判断

間取り→3Dパイプラインでは既にTripo APIをスキルに組み込み済み。コスト面でもTripoが有利で、GLBのみの出力制限もBlenderワークフローでは問題にならない。Tripoを継続利用するのが合理的。
