# PoC Step 6: 3Dレンダリング結果とユーザーのイメージ画像からテクスチャ・雰囲気が反映されたリアルな部屋映像を生成できる

> [ロードマップに戻る](./roadmap.md)

## 検証目標

Step 5のレンダリング画像（テクスチャなし）とユーザーのイメージ画像（1枚）を入力として、イメージの雰囲気・テクスチャが反映されたリアルな静止画像を生成できる。

---

## このステップの役割

```
Step 5の出力（レンダリング画像・テクスチャなし）
    ×
ユーザーのイメージ画像（雰囲気の基準・1枚）
    ↓
AIがテクスチャ・雰囲気を合成
    ↓
リアルな部屋の静止画像を出力
```

このステップがパイプラインの最終出力であり、ユーザーが「いいな」と思う雰囲気を実現する核心部分。

## 拡張方針（PoCスコープ外）

PoCではイメージ画像1枚のみを検証する。その後以下の順で拡張していく。

```
A. イメージ画像1枚（PoCスコープ）
    ↓
B. 複数枚のミックス
    ↓
C. 複数枚 + 詳細な指示
```

---

## 入力

| 入力                   | 内容                                                                           |
| ---------------------- | ------------------------------------------------------------------------------ |
| レンダリング画像       | Step 5の出力。空間の形状・アセット配置が正確に反映されているが、テクスチャなし |
| ユーザーのイメージ画像 | 1枚。雰囲気・テクスチャの基準となる画像（別部屋の写真・インテリア雑誌など）    |

---

## 検証ステップ

### Step 1: イメージ画像の雰囲気がレンダリング結果に反映されるか

- [ ] レンダリング画像とイメージ画像（1枚）を入力として与える
- [ ] AIがテクスチャ・雰囲気を合成した静止画像を出力できる
- [ ] 出力結果を目視で確認する
  - イメージ画像の雰囲気（色調・質感・照明）が反映されているか
  - Step 5のレンダリング結果の空間構造（形状・アセット配置）が維持されているか

---

## 合格基準

| 評価項目           | 合格の目安                                         |
| ------------------ | -------------------------------------------------- |
| 雰囲気の反映       | イメージ画像の色調・質感・照明が概ね反映されている |
| 空間構造の維持     | Step 5の形状・アセット配置が崩れていない           |
| 次ステップへの接続 | 出力画像がエンドツーエンド検証に渡せる             |

---

## このステップとDoc1・Doc2の接続

本ステップは既存パイプラインの課題と直接対応する。

| 既存課題                                    | 本ステップでの解決                                                                       |
| ------------------------------------------- | ---------------------------------------------------------------------------------------- |
| テキスト指示による画角変更が不安定（Doc 2） | 3Dレンダリングで空間構造を確定させた上でAIに渡すため、テキストで画角を指示する必要がない |
| camera_workと環境画像の画角不整合（Doc 1）  | カメラ位置はStep 5で3D空間上で正確に指定済み。AIはテクスチャ適用のみに集中できる         |
