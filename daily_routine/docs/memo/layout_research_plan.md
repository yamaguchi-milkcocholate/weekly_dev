# レイアウト提案の最新動向調査計画

## 背景

PoC3でClaude Codeによるレイアウト提案を検証した結果、以下が判明した:
- リアル制約のデジタル化（3D→SVG→構造化データ）は達成
- しかしAIによるレイアウト創造の品質が不足（人がどう動くかを想像できない）
- 核心の課題は「リアル制約をどのように定義してAIに渡すとうまく認識できるか」

この課題を解決するために、レイアウト提案・空間推論の最新動向を調査する。

## 調査目的

1. AIがリアル制約を認識してレイアウトを提案する最新手法を把握する
2. 特に「人間の生活シミュレーション」「動線の理解」をAIがどう実現しているかを調べる
3. 自プロジェクトに適用可能なアプローチを特定する

## 調査観点

### 観点1: 空間制約のAIへの入力方法

PoC3では構造化データ（JSON）とSVG画像を使ったが、最新手法ではどのような形式で空間をAIに渡しているか。

- テキスト（プロンプト）で渡す
- 画像（平面図、レンダリング画像）で渡す
- 3Dデータ（点群、メッシュ）を直接入力する
- シーングラフ（オブジェクト間の関係性）で渡す
- 上記の組み合わせ

### 観点2: 動線・生活シナリオの表現

PoC3の最大の弱点。先行研究ではどう扱っているか。

- 動線を経路として定義する手法
- 生活シナリオ（アクティビティシーケンス）をレイアウトに反映する手法
- 人間のエルゴノミクス（リーチ距離、通行幅）を制約に組み込む手法
- 家具の使用シーン（開閉、着席、通行）をシミュレーションする手法

### 観点3: 画像認識ベースの評価

PoC3で「SVGを見て評価する」アプローチを試みたが不十分だった。

- VLM（Vision-Language Model）によるレイアウト評価の事例
- レンダリング画像からの自動品質評価
- 人間の感覚（圧迫感、開放感、ゾーニング）をAIが判断する手法
- Gemini、GPT-4V等のマルチモーダルモデルの空間推論能力の比較

### 観点4: 最新のレイアウト生成手法（2025〜2026）

PoC3の旧技術調査（2024年時点）からの進化。

- LLM/VLMベースの最新レイアウト生成手法
- 拡散モデルベースのシーン合成の進化
- エージェント型アプローチ（反復改善、マルチエージェント）
- 商用ツール・APIの最新状況

### 観点5: リアル制約とAI創造の掛け算の事例

ビジョン「リアル制約 × AI創造」に直接関連する事例。

- 実際の間取り制約を入力としてAIがデザインを生成する商用サービス
- 建築・インテリア業界でのAI活用事例
- リアルデータ（3Dスキャン、BIM）からのAIレイアウト生成

## 調査方法

- 学術論文（arXiv, CVPR/ICCV/ECCV 2025-2026, SIGGRAPH 2025-2026）
- GitHub（公開実装）
- 商用サービス・API
- 技術ブログ・デモ

## 調査結果の活用

調査結果をもとに、PoC3の次のアプローチを決定する:
- どの入力形式で空間制約をAIに渡すか
- 動線・生活シナリオをどう表現するか
- どのモデル（Claude/Gemini/専用モデル）を使うか
- refineループのどこを強化するか
