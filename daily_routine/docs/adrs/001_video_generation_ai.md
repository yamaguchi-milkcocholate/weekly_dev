# ADR-001: 動画生成AIの選定

## ステータス

採用

## コンテキスト

Visual レイヤーで Image-to-Video 変換を行うAIを選定する必要がある。リファレンス画像（キャラクター画像）を入力として動画を生成するため、キャラクター同一性の維持が最重要の選定基準となる。

T0-2（動画生成AI比較検証）として、4種のAIに同一リファレンス画像を入力し、Gemini（gemini-2.5-flash）によるフレーム単位の自動評価で定量比較した。

## 候補

### Google Veo 2（Vertex AI）

- **総合同一性スコア:** 9.5/10（1位）
- **安定性（stdev）:** 0.53（最も安定）
- **生成時間:** 36.3秒（最速）
- **強み:** 全評価項目（顔9.62、髪9.75、服装9.38、体型9.25）で最高スコア。フレーム間のブレが極めて小さい
- **弱み:** Vertex AI 経由のみ利用可能。コストが高い（$0.50/秒、8秒で$4.00）
- **入力方式:** base64エンコード画像

### Runway Gen-4 Turbo

- **総合同一性スコア:** 8.5/10（2位）
- **安定性（stdev）:** 0.85
- **生成時間:** 75.6秒
- **強み:** 服装一貫性（9.5）と髪型一貫性（9.4）が高い。コストが安い（$0.05/秒、10秒で$0.50）
- **弱み:** 顔の類似度がやや低い（7.9）
- **入力方式:** URL指定

### Luma Dream Machine（Ray 2）

- **総合同一性スコア:** 7.2/10（3位）
- **安定性（stdev）:** 3.49（最も不安定）
- **生成時間:** 102.7秒（最遅）
- **強み:** 服装の一貫性は良好（9.0）
- **弱み:** 一部フレームで同一性が大きく崩れる（最低スコア1）。顔の類似度（6.6）と体型の一貫性（7.0）に課題
- **入力方式:** URL指定（keyframes.frame0）

### Kling AI v2.5 Turbo

- **評価:** クレジット不足のため未実施
- **認証方式:** Access Key + Secret Key による JWT（HS256）認証

## 決定

**Runway Gen-4 Turbo をデフォルトとして採用する。高品質代替として Veo を切替可能とする。Luma は除外する。**

理由は以下の通り。

1. **コスト効率:** Runway のコストは $0.05/秒（10秒で$0.50）であり、Veo の約1/8。初期フェーズではイテレーション回数が多くなるため、低コストでの試行が重要
2. **実用上十分な品質:** Runway の同一性スコア 8.5/10 は実用水準を満たす。服装一貫性（9.5）と髪型一貫性（9.4）が特に高い
3. **Veo の高品質代替価値:** PoC 時点の Veo 2 は全項目で最高スコア（同一性 9.5/10、stdev=0.53）であり、品質重視の本番運用時に切り替え可能
4. **生成速度:** Runway は 75.6秒で Veo 2（36.3秒）より遅いが、パイプライン全体のボトルネックにはならない
5. **Luma の除外:** 安定性が著しく低い（stdev=3.49、最低スコア1）ため、本番パイプラインでの使用には適さない

**Veo バージョン更新:** PoC は Veo 2 で実施したが、高品質代替として採用するバージョンは Veo 3（`veo-3.0-generate-001`）とする。Veo 3 は Veo 2 の後継であり、品質・速度の改善が見込まれる。デフォルトは引き続き Runway Gen-4 Turbo であり、Veo 3 は設定切り替えによるオプション利用とする。

## 結果

- Visual レイヤーの**デフォルト**動画生成AIとして Runway Gen-4 Turbo（`gen4_turbo`）を使用する
- 設定で Veo 3（`veo-3.0-generate-001`、Vertex AI 経由）に切り替え可能とする（高品質代替）
- Kling はクレジット追加後に再評価の余地を残すが、現時点では未採用
- Luma は不採用とし、実装対象から除外する
- コスト見積もり: Runway 10秒動画1本あたり $0.50、Veo 3 8秒動画1本あたり $4.00
