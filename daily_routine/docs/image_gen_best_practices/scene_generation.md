# Scene Generation（シーン生成 — キャラ × 環境融合）

## 推奨手法

- **推奨 API**: Gemini 3 Pro Image (`gemini-3-pro-image-preview`) + Flash (`gemini-3-flash-preview`)
- **方式**: 2パス — Flash 最小指示分析 → Pro シーン生成（C3-I1）
- **コスト**: $0.05/シーン（Flash 1回 $0.01 + Pro 1回 $0.04）
- **入力**: キャラクター画像 + 環境画像 + Identity Block + ポーズ指示

---

## 用途別の推奨パターン

| 用途               | 推奨パターン | 方式                                | コスト    |
| ------------------ | ------------ | ----------------------------------- | --------- |
| 画像入力型（基本） | **C3-I1**    | Flash 最小指示分析 → Pro シーン生成 | **$0.05** |
| テキスト環境型     | C3-T         | Pro 直接生成（環境画像なし）        | $0.04     |

**本番パイプラインでは C3-I1 を推奨**。最小指示により Pro が画像参照を主体に自然な合成を行い、空間認識・接地感・ポーズの自然さが最良。

---

## C3-I1 vs C3-I2: なぜ最小指示が最良か

| 観点               | C3-I1（最小指示）                     | C3-I2（テキストリッチ）                     |
| ------------------ | ------------------------------------- | ------------------------------------------- |
| Flash プロンプト長 | **~53 words（簡潔）**                 | ~136 words（約2.5倍）                       |
| Pro の自由度       | **高い** — 画像参照を主体に自然な合成 | 低い — テキスト制約を満たすことが優先       |
| 空間認識・接地感   | **安定** — 全ポーズで自然             | **動的ポーズで不安定** — walking で足が浮く |
| キャラクター一貫性 | 良好                                  | 良好                                        |
| 環境の再現         | **良好** — 環境画像を直接参照         | 良好                                        |

C3-I2 は Flash に環境記述テキスト + シナリオコンテキストを追加で渡す。Flash はこれらを取り込んで ~136 words の長文プロンプトを生成する。Pro はこの全てのテキスト制約を同時に満たそうとするため:

1. **テキスト指示に引っ張られ、画像参照への依存度が下がる**
2. **空間認識・接地感が犠牲になる** — 服装ディテールや照明条件の記述が優先される
3. **動的ポーズほど悪影響が出る** — walking で身体が硬く、足が浮いて見える

**B-1 の S3M 知見（最小指示メタプロンプトが最良）と完全に一致**。

### テキスト情報の投入判断

| Flash への追加情報   | C3-I1    | C3-I2 | 効果                           |
| -------------------- | -------- | ----- | ------------------------------ |
| Identity Block       | **あり** | あり  | **有効**（キャラ特定に寄与）   |
| 環境記述テキスト     | なし     | あり  | **逆効果**（プロンプト肥大化） |
| シナリオコンテキスト | なし     | あり  | **逆効果**（プロンプト肥大化） |

Identity Block はキャラクター特定に有用。環境記述テキストとシナリオコンテキストは環境画像と重複する情報であり、プロンプトを不必要に膨張させる。

## プロンプトテンプレート

### Flash 最小指示分析（C3-I1 Step 1）— 推奨

キャラ画像 + 環境画像 + 最小限のテキスト情報を分析し、シーンプロンプトを生成する。

```
Analyze both images carefully.
Image 1 shows the character. Image 2 shows the environment.
Generate an image generation prompt that places the character
naturally in this environment.
The character is: {Identity Block}
The character's pose: {ポーズ指示}
Output only the prompt text, nothing else.
```

- **temperature=0.0** で決定的出力にする
- Identity Block は C1-ID（生成後の画像分析）から取得
- 環境記述テキスト・シナリオコンテキストは**渡さない**

### Pro シーン生成（C3-I1 Step 2）

Flash 生成プロンプト + キャラ画像 + 環境画像でシーンを生成する。

```
Image 1 shows the character reference. Image 2 shows the environment reference.
{Flash生成プロンプト}
Single person only, solo. Photo-realistic, natural lighting.
```

### テキスト環境型（C3-T）

環境画像が不要な場合に使用する。

```
Image 1 shows the character: {Identity Block}
Generate a photo of this character in the following environment:
{環境記述テキスト}
The character's pose: {ポーズ指示}
Single person only, solo. Photo-realistic, natural lighting.
```

## ポーズ指示の設計

### 検証済みポーズ

| ポーズキー         | 指示文                                                             | 構図への影響                 |
| ------------------ | ------------------------------------------------------------------ | ---------------------------- |
| standing_confident | `standing confidently with one hand on hip, looking at the camera` | 全身中距離ショット           |
| walking            | `walking towards the camera with a natural stride`                 | 全身中距離ショット           |
| selfie             | `taking a selfie, smiling`                                         | 上半身近接アングルに自動調整 |

### ポーズ指示のガイドライン

- **全身が必要な場合**: ポーズ指示に `full body from head to feet` を追加
- **selfie ポーズ**: Pro がカメラアングルを自撮り風に自動調整する。全身が不要なシーン（SNS投稿風など）に適する
- **ポーズとアクセサリーの整合性**: ヘルメットを持っているキャラクターの場合、ポーズによってヘルメットの配置が自動調整される（腕に抱える、テーブルに置くなど）

### selfie ポーズのベストプラクティス

selfie の記述方法によって Pro の生成視点が大きく変わる。

| 記述方法                                   | Pro の解釈                                  | 結果                                                       |
| ------------------------------------------ | ------------------------------------------- | ---------------------------------------------------------- |
| `takes a selfie` / `taking a selfie`       | **内カメラ視点** — カメラ＝スマホ           | スマホが映らず、実際の自撮り写真のような自然な構図         |
| `holding up a smartphone to take a selfie` | **第三者視点** — スマホを持つ人を外から撮影 | スマホが手に映り込み、自撮りしている人を第三者が撮った構図 |

**推奨**: selfie ポーズ指示では `taking a selfie` と簡潔に書き、**`holding up a phone/smartphone` とデバイスを明示しない**。デバイスを明示すると Pro が「道具を持つ人物の描写」モードに切り替わり、内カメラ視点ではなく第三者視点になる。

これは C3-I1 の最小指示知見と同一の原理: **具体的なオブジェクト指示の追加が Pro の解釈モードを意図しない方向に切り替える**。

## 入力パイプライン

### C1 → C2 → C3 の全体フロー

```
C1-F2-MA: キャラクター生成（$0.13/clothing）
  出力: 正面・側面・背面の参照画像セット
    ↓
C1-ID: Identity Block 抽出（$0.01）
  出力: キャラクターの詳細テキスト記述
    ↓
C2-R2: 環境生成（$0.04/環境）
  出力: 人物不在の環境画像
    ↓
C3-I1: シーン融合（$0.05/シーン）
```

**注意**: C2-ED（環境記述テキスト抽出）は C3-I1 では不要。C3-T（テキスト環境型）を使用する場合のみ必要。

## 環境画像の有無による差

| 環境入力              | 環境構造の忠実さ               | 推奨場面                       | コスト |
| --------------------- | ------------------------------ | ------------------------------ | ------ |
| **画像あり（C3-I1）** | **忠実**（構造物・構図が一致） | 参照写真の環境を再現したい場合 | $0.05  |
| テキストのみ（C3-T）  | 概念は合うが構造が異なる       | 特定の環境写真がない場合       | $0.04  |

**具体例**: ダイビングボートの環境画像がある場合

- C3-I1: ボートのデッキ構造（屋根、ベンチ）、島の配置がC2出力と一致
- C3-T: 「ボート＋海」は合っているが、船首に立つ構図に変わるなど構造が異なる

## 注意点・アンチパターン

| アンチパターン                                             | 症状                                                     | 対策                                                        |
| ---------------------------------------------------------- | -------------------------------------------------------- | ----------------------------------------------------------- |
| Flash に環境記述テキスト・シナリオコンテキストを渡す       | **Pro が過制約になり空間認識・接地感が低下**             | Flash には画像 + Identity Block + ポーズのみ渡す（C3-I1）   |
| selfie ポーズで全身を期待                                  | 上半身のみの近接アングルになる                           | `full body from head to feet` をポーズ指示に追加            |
| 環境画像なしで特定の環境構造を再現しようとする             | テキストだけでは構造的特徴が失われる                     | 環境画像を入力する（C3-I1）                                 |
| Flash 分析なしで直接 Pro にキャラ+環境画像を渡す           | Proが融合方法を自由に解釈、不安定                        | Flash 分析でシーンプロンプトを生成してからProに渡す         |
| Identity Block に生成前の Flash 記述を使用                 | 実際の生成結果との乖離                                   | C1-ID（生成後の画像分析）から取得した Identity Block を使用 |
| selfie ポーズで `holding up a smartphone` とデバイスを明示 | **第三者視点になる**（スマホを持つ人を外から撮った構図） | `taking a selfie` とだけ書き、デバイスを明示しない          |

## C1・C2 知見との整合性

### 1パス vs 2パス の判断基準（C1・C2・C3 統合）

| タスク             | 入力              | 最適      | 理由                              |
| ------------------ | ----------------- | --------- | --------------------------------- |
| C1: 単一参照キャラ | 人物画像1枚       | **1パス** | Flash の自由提案が発散するリスク  |
| C1: 融合キャラ     | 人物+服装 2枚以上 | **2パス** | 複数画像の融合をFlashが言語化     |
| C2: 環境再現       | 環境写真1枚       | **1パス** | テキスト化で細部が失われる        |
| C3: シーン融合     | キャラ+環境 2枚   | **2パス** | 複数要素の融合指示をFlashが最適化 |

**一般化**: 「入力画像1枚で同じものを再現」は1パス、「複数画像の融合・配置」は2パスが安定。C3はキャラ+環境の2要素融合タスクなので2パスが有効。

### Flash への情報量の最適化（C1・C3 統合）

| タスク                 | Flash に渡す情報                     | 結果                     |
| ---------------------- | ------------------------------------ | ------------------------ |
| C1: 単一参照→摂動      | 自由提案を許可                       | **発散**（髪色変更など） |
| C3: テキストリッチ融合 | Identity Block + 環境記述 + シナリオ | **過制約**（接地感低下） |
| C1: 融合キャラ         | 画像 + 最小指示                      | **安定**                 |
| C3: 最小指示融合       | 画像 + Identity Block + ポーズ       | **安定**                 |

**一般化**: Flash には「画像 + 最小限のテキスト」を渡すのが最適。テキスト情報の追加は品質向上に寄与せず、プロンプト肥大化による過制約のリスクがある。

## 検証エビデンス

### Phase C-3: シーン融合検証（2026-03-02）

- **モデル**: `gemini-3-pro-image-preview`（画像生成）、`gemini-3-flash-preview`（テキスト分析）
- **方式**: Python SDK（`google-genai`）、`ImageConfig(aspect_ratio="9:16")`
- **検証パターン**: 3種（C3-I1, C3-I2, C3-T）
- **入力**: C1-F2-MA 正面画像（clothing_4）+ C2-R2 環境画像（env_1, env_2）
- **ポーズ**: 3種（standing_confident, walking, selfie）
- **生成数**: 18枚（3パターン × 2env × 3ポーズ）、**コスト**: $0.84

#### 主要な発見

1. **最小指示 Flash 分析（C3-I1）が最良**: Pro が画像参照を主体に自然な合成を行い、空間認識・接地感・ポーズの自然さが最良
2. **テキストリッチ Flash 分析（C3-I2）は過制約を引き起こす**: 環境記述テキスト + シナリオコンテキストの追加で Flash プロンプトが ~136 words に膨張し、Pro の自由度が低下。動的ポーズ（walking）で足が浮く・身体が硬い等の不自然さが発生
3. **環境画像が構造的忠実さに重要**: テキストのみ（C3-T）では概念は合うが構造が異なる。環境画像があると構造物・構図が一致
4. **ポーズ指示がカメラ構図にも影響**: selfie ポーズでは Pro が自動的に自撮り風アングルに調整
5. **C1・C2 の生成物を入力にしても品質を維持**: 生成画像→生成画像の連鎖でも劣化なし
6. **B-1 S3M 知見を再確認**: 最小指示メタプロンプトがシーン融合でも最良
