# 画像生成AI比較検証 設計書

## 1. 概要

- **対応する仕様書セクション:** 3.4章（Asset Generator）
- **対応するサブタスク:** T0-3（画像生成AI比較検証）
- **このサブタスクで実現すること:**
  - 画像生成AI 3種（Stability AI API / DALL-E 3 / Gemini）を同一条件で比較評価する
  - キャラクター一貫性・画質・API利便性の観点から、Asset Generatorに最適なAIを選定する
  - 評価結果をADR `adrs/002_image_generation_ai.md` に記録する

## 2. スコープ

### 対象範囲

- 3つの画像生成AIに対する統一的な評価プロトコルの設計と実行
- テストキャラクターの定義と各AIへの最適化プロンプト作成
- 人間による目視評価 + GPT-4o Visionによる定量評価の併用
- 評価レポートの生成とADRへの記録

### 対象外

- Asset Generatorモジュールの本格実装（T1-3で実施）
- 小物・背景の画像生成評価（本検証はキャラクター一貫性に焦点）
- 動画生成AIとの連携検証（T0-2で実施）

## 3. 技術設計

### 3.1 検証対象AI

| AI           | モデル/バージョン         | アクセス方法                  | 備考                                         |
| ------------ | ------------------------- | ----------------------------- | -------------------------------------------- |
| Stability AI | SD3.5 Large (`sd3.5-large`) | REST API (`api.stability.ai/v2beta`) | 画像生成特化、ネガティブプロンプト・シード固定に対応 |
| DALL-E 3     | dall-e-3                  | OpenAI API                    | 自然言語プロンプトへの追従性が高い           |
| Gemini       | Gemini 3 Pro Image (`gemini-3-pro-image-preview`) | Google AI API (`google-generativeai` SDK) | 高度な推論（思考モード）による画像生成。日本語テキスト描画、最大14枚の参照画像による編集に対応 |

### 3.2 テストキャラクター仕様

評価の公平性を担保するため、以下の特徴を持つ1体のテストキャラクターを定義する。特徴が具体的であるほど一貫性の評価がしやすくなる。

```yaml
character:
  name: "テストキャラクター Aoi"
  gender: 女性
  age_appearance: 25歳前後
  hair:
    color: ダークブラウン
    style: セミロング、毛先が軽く内巻き
  eyes:
    color: ブラウン
    shape: やや大きめのアーモンド型
  skin: 明るい肌色
  build: 標準体型、やや細身
  outfit:
    top: 白いブラウス（襟付き）
    bottom: ネイビーのタイトスカート（膝丈）
    shoes: ベージュのパンプス
    accessories: 小さなゴールドのピアス、腕時計
  style: "リアル寄りのセミリアリスティック（アニメ調ではない）"
```

### 3.3 生成画像セット（各AIごと3枚）

仕様書3.4章のAsset Generator要件（正面・横・背面の基本ポーズ）に対応する3枚を生成する。

| #   | ビュー                      | 目的               | 評価ポイント                         |
| --- | --------------------------- | ------------------ | ------------------------------------ |
| 1   | 正面全身                    | 基本リファレンス   | 顔の詳細、服装の再現性               |
| 2   | 横向き（右向き）上半身      | 角度変化での一貫性 | プロフィール時の顔の一貫性、髪の表現 |
| 3   | 斜め後ろ（左後方から） 全身 | 背面方向での一貫性 | 体型・服装・髪の背面表現             |

### 3.4 プロンプト設計

各AIの特性に合わせてプロンプトを最適化しつつ、伝達する情報は統一する。

#### 共通プロンプト要素（各AIに合わせて表現を調整）

```
キャラクター: 25歳の日本人女性、ダークブラウンのセミロングヘア（毛先内巻き）、
ブラウンのアーモンド型の目、白い襟付きブラウス、ネイビーのタイトスカート（膝丈）、
ベージュのパンプス、小さなゴールドピアス、腕時計
スタイル: セミリアリスティック、高品質、スタジオライティング
背景: 無地の白背景
```

#### AI別プロンプト調整方針

- **Stability AI API**: カンマ区切りのタグ形式 + ネガティブプロンプトを活用
- **DALL-E 3**: 自然言語での詳細な描写。リビジョン機能を考慮
- **Gemini**: 自然言語での詳細な描写。日本語プロンプトも検証

### 3.5 評価方法

#### 3.5.1 人間による目視評価（主評価）

以下の項目を1〜5のスケールで評価する。

| 評価項目         | 重み | 評価基準                                    |
| ---------------- | ---- | ------------------------------------------- |
| 顔の一貫性       | 30%  | 3枚間で同一人物として認識できるか           |
| 服装の一貫性     | 20%  | 服装の色・形・ディテールが3枚間で一致するか |
| 画質             | 20%  | 解像度、ディテール、アーティファクトの有無  |
| プロンプト追従性 | 15%  | 指定した特徴が正確に反映されているか        |
| スタイルの統一感 | 15%  | 3枚間で画風・トーンが統一されているか       |

**スコアリング基準:**

| スコア | 定義                                     |
| ------ | ---------------------------------------- |
| 5      | 完璧。プロのイラストレーターの品質に匹敵 |
| 4      | 優秀。わずかな差異はあるが実用上問題なし |
| 3      | 合格。目立つ差異はあるが許容範囲内       |
| 2      | 不十分。一貫性や品質に明確な問題あり     |
| 1      | 不可。パイプラインでの使用に耐えない     |

#### 3.5.2 AIによる定量評価（補助評価）

GPT-4o Visionを使い、生成された3枚の画像セットの一貫性を自動スコアリングする。

**評価フロー:**

1. 各AIで生成した3枚の画像を1つのプロンプトでGPT-4o Visionに入力
2. 以下の観点でJSON形式のスコアを出力させる:
   - `facial_consistency`: 顔の一貫性 (0-100)
   - `outfit_consistency`: 服装の一貫性 (0-100)
   - `style_consistency`: 画風の一貫性 (0-100)
   - `overall_quality`: 総合品質 (0-100)
   - `reasoning`: 評価理由（テキスト）
3. 各AIの結果を比較表にまとめる

### 3.6 API利便性の評価

プログラム的な観点での評価も実施する。

| 評価項目       | 評価内容                                     |
| -------------- | -------------------------------------------- |
| API設計        | RESTful / SDK提供有無、ドキュメント品質      |
| レスポンス時間 | 画像1枚あたりの生成時間                      |
| コスト         | 1画像あたりのAPI料金                         |
| 出力制御       | 解像度指定、シード固定、ネガティブプロンプト |
| レート制限     | 同時リクエスト数、月間制限                   |
| Python SDK     | 公式SDKの有無と成熟度                        |

### 3.8 APIクライアントインターフェース

各AIクライアントは共通のインターフェースに従う。

```python
from abc import ABC, abstractmethod
from pathlib import Path
from pydantic import BaseModel


class GenerationRequest(BaseModel):
    """画像生成リクエスト"""
    prompt: str
    negative_prompt: str | None = None
    width: int = 1024
    height: int = 1024
    seed: int | None = None


class GenerationResult(BaseModel):
    """画像生成結果"""
    image_path: Path
    generation_time_sec: float
    model_name: str
    cost_usd: float | None = None
    metadata: dict = {}


class ImageGeneratorClient(ABC):
    """画像生成AIクライアントの共通インターフェース"""

    @abstractmethod
    async def generate(self, request: GenerationRequest) -> GenerationResult:
        """画像を生成して保存し、結果を返す"""
        ...

    @abstractmethod
    def get_api_info(self) -> dict:
        """API情報（レート制限、コスト等）を返す"""
        ...
```

## 4. 実装計画

### ステップ1: 検証用モジュール構成の作成

- `poc/image_gen/` ディレクトリ構造を作成
- 共通インターフェース (`ImageGeneratorClient`) を実装
- テストキャラクターのプロンプト定義 (`config.py`)
- **完了条件:** モジュール構造が作成され、スクリプトが実行可能

### ステップ2: 各AIクライアントの実装

- Stability AI APIクライアント（`stability.py`）
- DALL-E 3クライアント（`dalle.py`）
- Gemini画像生成クライアント（`gemini.py`）
- 各クライアントで1枚テスト生成し、APIアクセスが正常に動作することを確認
- **完了条件:** 各APIで画像が1枚生成でき、ファイルに保存される

### ステップ3: 画像生成の実行（3AI x 3枚 = 9枚）

- 統一されたプロンプトで各AIから3枚ずつ画像を生成
- 生成時間・コストを記録
- **完了条件:** 9枚の画像が所定のディレクトリに保存される

### ステップ4: GPT-4o Visionによる自動評価

- 評価ロジック（`evaluate.py`）を実装
- 各AIの3枚セットをGPT-4o Visionに入力し、一貫性スコアを取得
- 結果を `ai_evaluation.json` に保存
- **完了条件:** 3つのAIの評価スコアがJSON形式で出力される

### ステップ5: 人間による目視評価

- 生成された9枚の画像を確認
- 3.5.1のスコアリング基準に従い評価
- API利便性の評価も実施

### ステップ6: 総合評価レポート + ADR作成

- 目視評価とAI評価の結果を統合
- 総合評価レポートを `report.json` に出力
- ADR `docs/adrs/002_image_generation_ai.md` を作成
- **完了条件:** ADRが作成され、採用AIが決定される

## 5. テスト方針

- **APIクライアントテスト:** 各クライアントのモック化されたユニットテスト（実API呼び出しなし）
  - リクエスト生成の正確性
  - レスポンスパースの正確性
  - エラーハンドリング
- **評価ロジックテスト:** GPT-4o Vision評価のプロンプト生成とレスポンスパースのテスト
- **統合テスト（手動）:** 実際にAPIを呼び出して画像を生成し、評価する（CI対象外）
