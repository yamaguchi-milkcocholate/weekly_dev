# Phase 2: 表面と光 ― 見た目を作る力

**目標:** 地形にリアルなマテリアルを適用し、ライティングでフォトリアルな見た目にできる。

---

## 2-1. Shader Editor と Principled BSDF `[実験済]`

### Shader Editorの開き方
- 画面下部のエディタタイプを「シェーダーエディター」に切り替え
- または「Shading」ワークスペースタブ

### 基本構造
- デフォルトで「Principled BSDF」→「マテリアル出力」の2ノードがある
- Principled BSDFにノードを接続して見た目を制御する

### 海のマテリアル設定 `[実験済]`

| パラメータ | 値 | 効果 |
|-----------|-----|------|
| ベースカラー | 深い青（`#0A3055`） | 水の色 |
| 粗さ（Roughness） | `0.05` | ツルツルの反射 |
| 伝播（Transmission） | `1.0` | 透明感 |
| IOR | `1.33` | 水の屈折率 |

## 2-2. 高度ベースマテリアル `[実験済]`

### ノード構成

```
テクスチャ座標 [オブジェクト] → XYZ分離 [ベクトル]
XYZ分離 [Z] → カラーランプ [係数]
カラーランプ [カラー] → Principled BSDF [ベースカラー]
```

### 仕組み
- テクスチャ座標: 頂点の位置情報を取得
- XYZ分離: Z値（高さ）だけを取り出す
- カラーランプ: Z値に応じて色をグラデーションで割り当て

### カラーランプの設定（亜熱帯の島）
- 左（低い）: 砂浜色（薄いベージュ `#C2B280`）
- 中間: 濃い緑（`#1A3A10`）
- 右（高い）: 緑（`#2D5A1E`）

## 2-4. Cyclesレンダリング `[実験済]`

- `F12` でレンダリング実行
- マテリアル未設定の状態ではレンダリングしても灰色/黒になるだけ
- **マテリアルプレビュー**モード（ヘッダー右上の球アイコン3番目）でリアルタイムに色を確認可能

### ビューポートシェーディングモード

| モード | 用途 |
|--------|------|
| ワイヤーフレーム | 辺だけ表示 |
| ソリッド | 形状確認（灰色） |
| **マテリアルプレビュー** | マテリアルの色をリアルタイム表示 |
| レンダープレビュー | レンダリングに近い表示 |

### 学び
- マテリアル設定前にレンダリングしても意味がない
- ソリッドモードで形状確認 → マテリアル設定 → マテリアルプレビューで確認 → Cyclesレンダリングが正しい順序
