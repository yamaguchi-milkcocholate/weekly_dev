# Phase 0: Blender基礎操作 学習ノート

**目標:** GUI操作に不自由しないレベルになる。3Dビューポートを自在に動かせる。
**完了条件:** Planeを追加 → Subdivision Surfaceモディファイアを適用 → Edit Modeで頂点を動かし → 保存できる

---

## 0-1. スタートアップシーンを理解する

Blenderを起動すると表示されるデフォルトシーンの構成要素:

| 要素 | 役割 |
|------|------|
| **Cube（グレーの立方体）** | メッシュオブジェクト。オレンジの輪郭 = 選択中 |
| **Light（同心円）** | シーンを照らす光源 |
| **Camera（三角錐）** | レンダリング時の視点 |
| **3D Cursor（赤白の十字）** | 新規オブジェクトの配置基点 |
| **Grid Floor（灰色の格子）** | 世界座標の床面。赤=X軸、緑=Y軸 |

**やること:** Blenderを起動し、各要素を目視で確認する。

## 0-2. 3Dビューポートのナビゲーション

### モード（最重要概念）

モードによってキーバインドもメニューも完全に変わる。**「操作できない」はモード違いが原因であることが多い。**

| モード | 用途 | 切り替え |
|--------|------|----------|
| **Object Mode** | オブジェクトの配置・移動 | デフォルト |
| **Edit Mode** | メッシュの頂点・辺・面の編集 | `Tab` |
| **Sculpt Mode** | ブラシで形状を彫る | `Ctrl-Tab` メニュー |

### ビューポート操作（MMB = ミドルマウスボタン）

| 操作 | ショートカット |
|------|--------------|
| **視点回転（オービット）** | `MMB` ドラッグ |
| **視点平行移動（パン）** | `Shift-MMB` ドラッグ |
| **ズーム** | ホイール |
| **ズーム限界時のDolly** | `Shift-Ctrl-MMB` ドラッグ |

### 視点切り替え（テンキー）

| キー | 視点 |
|------|------|
| `Numpad 1` | 正面（Front） |
| `Numpad 3` | 右面（Right） |
| `Numpad 7` | 上面（Top） |
| `Numpad 0` | カメラビュー |
| `Numpad 5` | 透視/平行投影 トグル |
| `Numpad .` | 選択オブジェクトを中心に表示 |
| `Home` | シーン全体を表示（**迷子になったらこれ**） |

### トラブルシューティング

| 症状 | 対処 |
|------|------|
| ズームできなくなった | `Shift-Ctrl-MMB` でDolly |
| オブジェクトが消えた | `Numpad /`（ローカルビュー）を再度押す |
| 迷子になった | `Home` で全体表示 |
| カメラビューに戻れない | `Numpad 0` |

**やること:** 適当なオブジェクトを配置し、オービット・パン・ズーム・視点切り替えを各10回以上繰り返す。

## 0-3. オブジェクトの追加・変形

### オブジェクトの2層構造

- **Object** = 位置・回転・スケール（変換情報）
- **Object Data** = メッシュの頂点など実体データ
- オリジン（黄色のドット）が変換の基準点

### 基本操作

| 操作 | ショートカット |
|------|--------------|
| 移動 (Grab) | `G` |
| 回転 | `R` |
| スケール | `S` |
| 軸を限定 | `G/R/S → X/Y/Z` |
| 数値入力 | `G → Z → 2 → Enter`（Z方向に2移動） |
| 全選択 | `A` |
| 全解除 | `Alt-A` |
| オブジェクト追加 | `Shift-A` |

### 選択の仕組み

- 最後に選択したオブジェクト = **アクティブオブジェクト**（黄色）
- その他の選択済み = 橙色
- `Shift-LMB` で追加選択

### コレクション（フォルダ的な整理）

- `M` でコレクションへ移動
- 1つのオブジェクトが複数コレクションに所属可能
- コレクションの「非表示」でレンダリングのオン/オフ制御

**重要な注意:**
- **Scaleを適用し忘れると後でモディファイアが狂う** → `Ctrl-A → All Transforms` を習慣にする

**やること:** Cube削除 → `Shift-A` でPlane追加 → `G`, `R`, `S` でそれぞれ操作 → 軸限定（`G → Z`等）を試す

## 0-4. Edit Mode基礎

### メッシュの3要素

| 要素 | 役割 |
|------|------|
| **頂点 (Vertex)** | 3D空間の点。最小単位 |
| **辺 (Edge)** | 2頂点を結ぶ直線 |
| **面 (Face)** | 表面。レンダリングで見える部分 |

### 面の種類

- **Quad（四角面）**: 基本的にこれを使う。SubDivに最適
- **Tri（三角面）**: 常に平面。計算が速いが避けるのが無難
- **N-gon（5角以上）**: なるべく避ける

### 選択モードの切り替え

| モード | ショートカット |
|--------|--------------|
| 頂点選択 | `1` |
| 辺選択 | `2` |
| 面選択 | `3` |
| ループ選択 | `Alt-LMB` |

### 主要な編集操作

| 操作 | ショートカット | 備考 |
|------|--------------|------|
| 押し出し | `E` | 最頻出 |
| インセット | `I` | 面の内側に枠 |
| 複製 | `Shift-D` | |
| 削除 | `X` | 穴が開く |
| 溶解 | `Ctrl-X` | 穴が開かない削除 |
| マージ | `M` | 重複頂点除去: `M → By Distance` |

### ハマりポイント

- **Extrude後にRMBキャンセルしても新しい面は残る**（移動のみキャンセル）
- **DeleteとDissolveは別物**。Deleteは穴が開く、Dissolveは開かない
- **X-Rayモードをオフのまま選択すると裏面の頂点を選択できない**

**やること:** Planeで`Tab`→Edit Mode → 頂点選択(`1`) → 頂点を`G`で動かす → 辺選択(`2`)に切り替え → 面選択(`3`)に切り替え

## 0-5. Modifier概念

### 核心概念

- **非破壊的**: 元のジオメトリを変えずに効果を適用
- **スタック順序**: 上から順に評価される。**順序が結果を変える**
- `Apply`（`Ctrl-A`）で確定 → 元に戻せなくなる（破壊的）

### 4カテゴリ（日本語UI）

| カテゴリ | 日本語UI | 役割 | 例 |
|----------|----------|------|-----|
| **Generate** | 生成 | ジオメトリの追加・変形 | サブディビジョンサーフェス, ミラー, ベベル |
| **Deform** | 変形 | 形状変更 | アーマチュア, ラティス, ディスプレイス |
| **Edit** | 編集 | データ操作 | 頂点ウェイト編集 |
| **Simulate** | 物理演算 | 物理シミュレーション | クロス, 流体 |

### スタック順序の鉄則

- `Mirror → SubDiv`: 正しく結合された滑らかなメッシュ
- `SubDiv → Mirror`: 継ぎ目が割れる（NG）
- **最上位から順にApplyする**。途中のモディファイアを先にApplyしない

**やること:** PlaneにSubdivision Surfaceモディファイアを追加 → Levelsを変えて効果を観察 → Edit Modeに切り替えて頂点を動かし、SubDivの効果を実感

## 0-6. ファイル操作

### 基本操作

| 操作 | ショートカット |
|------|--------------|
| 上書き保存 | `Ctrl-S` |
| 別名保存 | `Shift-Ctrl-S` |
| 開く | `Ctrl-O` |
| 最近開いたファイル | `Shift-Ctrl-O` |
| 連番保存 | `Ctrl-Alt-S` |

### 復元

- **Auto Save**: `File → Recover → Auto Save`（クラッシュ時）
  - 注意: Sculpt/Texture Paint/Edit Modeの変更は自動保存されない
- **Last Session**: `File → Recover → Last Session`（誤って閉じた時）

### 注意点

- `.blend`はすべてのデータを1ファイルに含むが、外部テクスチャは参照（リンク）
- ファイル移動時はテクスチャファイルも一緒に移動すること
