# Blender学習データソース

## 背景と目的

### なぜデータソースが必要か

2026-04-09のBlender地形生成実験で以下が判明:

- **Claude Codeは3DCGの羅針盤を持っていない**: Web検索で集めた技術を並べて「こうすればできる」と提案しても正確さ30-40%程度
- **Claude Codeが羅針盤を持つ方法**: ユーザーが教材（書籍・プロのワークフロー解説等）を渡せば、そこから適切な知識を抽出できる。プログラミングでいう「公式ドキュメントやオライリー本を渡して、必要な情報を引き出す」と同じ構造
- **教材の質 = Claude Codeの判断の質**

### 3DCG教育の特殊性

プログラミング教育との違い:

- プログラミング: 書籍・ドキュメント（テキスト）が主流 → Claude Codeが直接読める
- **3DCG: 動画チュートリアルが主流** → Claude Codeが直接処理できない

### Claude Codeが処理できる形式

| 形式                       | 処理可否     | 備考                                           |
| -------------------------- | ------------ | ---------------------------------------------- |
| テキスト記事               | **読める**   | ブログ、公式ドキュメント等                     |
| 書籍（PDF）                | **読める**   | 体系的知識のベスト                             |
| .blendファイル             | **読める**   | bpyで解析してノード構造等を理解                |
| 画像（スクリーンショット） | **読める**   | ノード接続図、設定画面等                       |
| **動画チュートリアル**     | **読めない** | 3DCG教育の主流。トランスクリプトがあれば読める |

### 求めるデータソースの条件

1. **体系的**: 断片的なTipsではなく、ワークフロー全体をカバー
2. **プロの検証済み**: 実際のプロが実務で使っている手法
3. **Blender 4.x〜5.0対応**: 古すぎるバージョンの情報は使えない
4. **環境アート寄り**: 自然地形・風景がメインターゲット（キャラモデリングではない）
5. **Claude Codeが処理できる形式**: テキスト・PDF・画像が望ましい。動画の場合はトランスクリプトか要約が必要

## ユーザーの学習目標

- Blenderを自分で使いこなせるようになる
- 頭の中の世界やAIで拡張されたアイデアを3DCGにする
- フィジカルAI領域への基盤技術として3DCGを習得する
- 学ぶ過程でClaude Codeで自動化・拡張できるポイントを見つける

## ユーザーのバックグラウンド

- コンピューターサイエンス大学院卒
- エンジニアリング・プログラミングスキルあり
- AIパイプライン構築経験あり（PoC 0-9で間取り図→3Dウォークスルー動画パイプライン構築済み）
- Blenderは初心者（基本操作から学ぶ必要あり）
- bpyスクリプトでの操作経験はあるが、GUIベースの実制作経験なし

## データソース管理

データソースはgitリポジトリとして管理されており、`datasource/` 配下にclone済み。
Claude Codeがテキストとして直接読み込める形式で保持する。
