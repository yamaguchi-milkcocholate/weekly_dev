# ビジョンと戦略

## 目的

家族・大切な人・自分に使える時間を増やすために、場所・時間に縛られない働き方を実現する。

## 事業の軸

**「リアル制約 × AI創造」の掛け算。**

- リアル空間の制約を忠実にデジタル化した上で、AIがその制約の中で創造・予測する
- 忠実な再現だけなら測量・CADと変わらない。AIの創造だけなら現実の制約を無視した絵空事になる
- **掛け合わせることで「現実に実装可能なアイデアが自動で生まれる」という唯一無二の価値を作る**

## 戦略

「デジタルだけで完結するスキルはAIに代替される」という認識のもと、**非構造データ（画像・3D・動画等）を扱うAIパイプラインを、業界横断で組み立てられる人材**になる。

データサイエンティストの経験を土台に、非構造データ × AI APIのパイプライン構築力を武器とする。

### 戦い方: 業界の手作業をエンジニアリング × AIで変革する

既存業界の専門家はドメイン知識を持つがエンジニアではない。そこに2つのレバレッジを掛ける:

1. **既存市場を観察し、専門家が手作業でデジタル成果物を作っている領域を見つける**
2. **2つの変革を仕掛ける**:
   - **納品物のエンジニアリング化**: 成果物をアプリ化・インタラクティブ化し、専門家には出せない形で届ける
   - **制作プロセスのAI自動化**: 手作業のワークフローをパイプラインで効率化する

#### 具体例: 建築パース市場

ココナラの建築パースカテゴリ（4,157件、累計8万件超の販売）を観察した結果:

- 出品者は建築士・インテリアコーディネーター。3DCGでモデルを作っているが、**納品物は静止画像**
- → **納品物の変革**: 3Dモデルを**Webベースの3Dビューワー**として納品。顧客が任意のアングルから閲覧可能に
- → **制作プロセスの変革**: 手作業の3DCG制作をAI + bpyスクリプトで自動化・効率化

この「観察 → レバレッジ発見 → 変革」のパターンを他の業界にも横展開する。

## スキル投資: 3DCG（Blender）

### なぜ3DCGを習得するか

3つの動機が重なっている:

1. **パイプライン構築の精度を上げるため**: パイプラインの中を流れるデータ（3D）の性質を理解していないと、質の高いパイプラインは設計できない。プログラミングでいえば「コードが書けないアーキテクトは設計ができない」のと同じ
2. **フィジカルAI領域への布石**: リアル空間の3D化はフィジカルAIの基盤技術。AIと関連してこれから伸びる領域
3. **個人の創作欲求**: 頭の中の世界や、生成AIで拡張されたアイデアを3DCGとして実現したい

### AI × 3DCGの構造

プログラミングの世界で起きていることと同じ構造:
- 素人 + AI = 変なものしか作れない
- **スキルがある人 + AI = 10倍の生産性**
- AIが不要にするのは「新規参入者の低い壁」であり、スキルを持つ人間はより拡張される

3DCGのスキルがあれば、AIツール（Claude Code等）で自動化・拡張できる。スキルがなければAIを使っても方向性が定まらない。

### 学習方針

- まずBlenderを自分で使いこなせるレベルになる
- 教材を用意し、Claude Codeに渡すことで知識抽出の精度を上げる
- 学ぶ過程でClaude Codeで自動化できるポイントを見つけ、スキル化する

## 成長パス

```
→ 既存市場を観察し、エンジニアリング × AIで変革できる領域を見つける
→ 発信（技術記事 + デモ）でインバウンド集客
→ 3DCGスキル習得（空間具現化の基盤投資）
→ 複数領域に関わる中で、自分が没頭できる事業テーマを探す
→ 独立
```

## 方向性の議論（2026-04-13）

### パイプラインの本質

間取りパイプラインに限定されない。本質は:

```
「まだ存在しない空間を、制約から具現化する」
```

- 2D/テキストの制約定義 → 3D空間構築 → AI映像化
- 入口を差し替えることで横展開が可能:

| 入口（制約定義） | 出力 | 対象 |
|----------------|------|------|
| 間取り図 | インテリア動画 | 不動産・建築 |
| 店舗コンセプト文 + 図面 | 店内イメージ動画 | 出店検討者 |
| イベント会場図 + テーマ | 会場演出プレビュー | イベント企画 |
| 商品写真 + 空間指定 | 商品の空間展示映像 | EC・D2C |

### 検討した方向性と結論

| 方向性 | 評価 | 理由 |
|--------|------|------|
| A. 間取りパイプラインを磨いて発信 | **採用（短期）** | PoC実績あり。発信でインバウンド集客 |
| B. Blender × AI創作 | **採用（A統合）** | ポートフォリオ作り + スキル習得として機能。「空間を具現化する力」のデモになる |
| C. Physical AI（Isaac Sim等） | **保留** | visionとの整合性は最高だが、NVIDIA GPU必須・市場未成熟・個人参入が困難 |
| D. 組み込み / IoT | **不採用** | ハードウェア経験ゼロからの距離が遠すぎる |
| E. リアル空間スキャン（LiDAR等） | **不採用** | 入口が重くなる。スキャンは既存物件のみ対象で「AI創造」側を制限する |

### 集客モデル: 営業ではなく発信

営業（売り込み）は向いていない。発信でクライアントから依頼が来るインバウンド型を目指す。
ただし**発信の題材が重要**: 業界の課題を解決するデモ（before/after等）を見せることで、「この人に頼みたい」を引き出す。

### 3DCGスキル投資の位置づけ

- Blenderでの空間構築力 = パイプラインの入力品質 = AI出力の安定性（情報密度の原則）
- 体系的な学習ではなく、作りたいもの・見せたいもの駆動で学ぶ
- AI創作（アニメシーン等）もポートフォリオとして「空間具現化の力」を示す手段になる

### 将来への布石

Physical AI（C）の市場が成熟した際に参入するため、USD / OpenUSDの基礎を並行で学ぶことは有効。
ただし現時点では優先度を下げ、A + Bで実績と収益を確保することを優先する。

### 懸念・未解決

- 「空間具現化パイプライン」の市場規模が不明。需要が顕在化していない可能性がある
- 純粋なコーディング能力はAI代替の最前線にある。パイプライン構築力がどこまで差別化になるかは時間との勝負
- 「誰の課題を解決するか」がまだ定まっていない（実績なしの段階では定められないのも事実）
