# アイデアメモ

## 3Dビューワー納品による差別化（2026-04-13）

### 背景

ココナラの「建築・パース作成・インテリア」カテゴリ（4,157件、累計8万件超の販売実績）を調査した結果、以下の構造が判明:

- 出品者の多くは建築士・インテリアコーディネーター
- 3DCGでモデルを作っているにもかかわらず、**静止画像で納品**している
- 追加アングルが欲しい場合は追加料金 + 再依頼が必要

### アイデア

3Dモデルを**Webベースの3Dビューワー**として納品し、顧客が任意の場所から自由に閲覧できるようにする。

```
現状の市場:  3Dモデル作成 → 特定アングルでレンダリング → 画像納品
提案:        3Dモデル作成 → Webベース3Dビューワーとして納品
```

### なぜ差別化になるか

- 競合は建築士・デザイナーでありエンジニアではない
- 3Dパースを作る技術はあるが、Webアプリとしてデプロイする技術がない
- **3DCG × エンジニアリングの掛け算**で、片方だけ持つ人には真似できない

### 技術構成

```
Blender → glTF/GLB出力 → Webビューワー（Three.js / model-viewer / React Three Fiber）
```

### 付加価値の可能性

- ライティング切り替え（朝/昼/夜）
- 家具の入れ替えシミュレーション
- VR対応（スマホでルームスケール体験）
- 共有URL発行（クライアントがそのまま施主に見せられる）

### visionとの整合性

```
リアル制約（間取り・図面） × AI創造（空間の具現化） × エンジニアリング（インタラクティブ体験）
```

### 検証方法

既存の間取りPoCの成果物をglTFエクスポート → model-viewerで表示するプロトタイプを作る。
